# 题目

https://www.acwing.com/problem/content/166/

给定一张 N 个点 M 条边的有向无环图，分别统计从每个点出发能够到达的点的数量。

#### 输入格式

第一行两个整数 N,M，接下来 M 行每行两个整数 x,y，表示从 x 到 y 的一条有向边。

#### 输出格式

输出共 N 行，表示每个点能够到达的点的数量。

#### 数据范围

1≤N,M≤30000,  
1≤x,y≤N

#### 输入样例：

```
10 10
3 8
2 3
2 5
5 9
5 9
2 3
3 9
4 8
2 10
4 9
```

#### 输出样例：

```
1
6
3
3
2
1
1
1
1
1
```

# 思路

先试试dfs,好TLE了

## 题目本质：统计每个点的可达节点数

给定一个 **有向无环图（DAG）**，对于每个节点 (i)，要求：

```text
从 i 出发可以到达多少个节点（包括自己）
```

例如：

```
1 → 2 → 4
 \ 
  → 3 → 4
```

可达集合：

```
1 → {1,2,3,4}
2 → {2,4}
3 → {3,4}
4 → {4}
```

输出每个节点的集合大小。

---

## 为什么不能对每个点做 BFS / DFS

最直接的方法：

```
每个节点跑一次 BFS
```

复杂度：

```
O(n × (n + m))
```

当：

```
n = 30000
```

时会达到约

```
9 × 10^8
```

明显超时。

因此必须 **复用已经计算过的可达信息**。

---

## 核心思想：可达集合的传递

如果存在边：

```
u → v
```

那么：

```
u 能到达的点
=
{u} ∪ v 能到达的点
```

也就是：

```
f[u] |= f[v]
```

含义：

```
把 v 的可达集合合并到 u
```

---

## 为什么需要拓扑排序

如果直接做集合转移，会遇到问题：

```
f[u] |= f[v]
```

但此时：

```
v 的可达集合可能还没计算
```

因此需要保证：

```
先计算终点
再计算前驱
```

解决方法就是 **拓扑排序**。

拓扑排序会得到：

```
q[0], q[1], ..., q[n-1]
```

表示从前驱到后继的顺序。

而 DP 时需要 **逆序处理**：

```
q[n-1] → q[0]
```

这样可以保证：

```
后继节点先计算完成
```

---

## bitset 的作用

代码中定义：

```cpp
bitset<N> f[N];
```

含义：

```
f[i][j] = 1
表示 i 可以到达 j
```

例如：

```
f[1] = {1,2,3,4}
```

bitset 的优势是：

```
集合合并可以用位运算
```

例如：

```
f[u] |= f[v]
```

时间复杂度为：

```
O(n / 64)
```

比普通数组快很多。

---

## 算法整体流程

整体步骤：

```
建图
↓
拓扑排序
↓
逆拓扑DP
↓
统计bitset
```

---

## 建图（邻接表）

使用邻接表存储图：

```cpp
int h[N], e[M], ne[M];
```

含义：

```
h[i]    i 的第一条边
e[k]    边的终点
ne[k]   下一条边
```

加边函数：

```cpp
void add(int a, int b)
{
    e[idx] = b;
    ne[idx] = h[a];
    h[a] = idx++;
}
```

表示：

```
a → b
```

---

## 拓扑排序

拓扑排序的作用：

```
生成 DAG 的拓扑序
```

步骤：

1. 统计每个点的入度
    
2. 将入度为 0 的点加入队列
    
3. 不断删除点并更新入度
    

最终得到：

```
q[0..n-1]
```

表示拓扑顺序。

---

## 逆拓扑 DP

DP 按 **逆拓扑序**进行：

```cpp
for (int i = n - 1; i >= 0; i--)
```

原因：

```
终点没有后继
最先计算
```

转移：

```
f[j][j] = 1
```

表示：

```
节点可以到达自己
```

然后合并后继节点：

```
f[j] |= f[v]
```

其中：

```
j → v
```

最终：

```
f[j] 包含所有可达节点
```

---

## 统计答案

bitset 提供函数：

```cpp
f[i].count()
```

表示：

```
集合中 1 的个数
```

也就是：

```
i 可以到达的节点数量
```

---

## 时间复杂度

拓扑排序：

```
O(n + m)
```

DP 转移：

```
n × (n / 64)
```

总复杂度：

```
O(n² / 64)
```

在：

```
n = 30000
```

时仍然可以通过。

---

## 典型套路总结

当题目满足：

```
DAG
+
需要统计可达节点
+
n 较大
```

常见解法就是：

```
拓扑排序
+
集合 DP
+
bitset 优化
```

记忆模型：

```
DAG 可达统计
=
拓扑序 + bitset 传递
```

# 代码

```c++
#include<bits/stdc++.h>
using namespace std;

// N：最大节点数
const int N=30010;

int n,m;

// 邻接表
int h[N],e[N],ne[N],idx;

// 入度数组
int d[N];

// f[i]：从 i 出发能到达哪些点（bitset 表示集合）
bitset<N> f[N];

// 存储拓扑序
vector<int> path;


// 加边 a -> b
void add(int a,int b){
    e[idx]=b;        // 边的终点
    ne[idx]=h[a];    // 下一条边
    h[a]=idx++;      // 更新头节点
}


// 拓扑排序（Kahn算法）
void topsort(){

    queue<int> q;

    // 找入度为0的点
    for(int i=1;i<=n;i++){
        if(d[i] == 0){
            q.push(i);
            path.push_back(i);   // 记录拓扑序
        }
    }

    // BFS拓扑排序
    while(!q.empty()){

        int u=q.front();
        q.pop();

        // 遍历 u 的所有出边
        for(int i=h[u];~i;i=ne[i]){

            int j=e[i];

            d[j]--;   // 删除边 u -> j

            // 如果入度变为0
            if(!d[j]){
                path.push_back(j);
                q.push(j);
            }
        }
    }
}


int main(){

    cin>>n>>m;

    // 初始化邻接表
    memset(h,-1,sizeof h);

    // 读入边
    while(m--){
        int a,b;
        cin>>a>>b;

        // a -> b
        d[b]++;       // b 入度 +1
        add(a,b);
    }

    // 拓扑排序
    topsort();


    // 逆拓扑序 DP
    // 从拓扑序的最后一个点开始
    for(int i=n-1;i>=0;i--){

        int u=path[i];

        // 自己可以到达自己
        f[u][u]=1;

        // 遍历所有 u -> j
        for(int i=h[u];~i;i=ne[i]){

            int j=e[i];

            // 集合合并
            // u 能到达的点 = j 能到达的点
            f[u] |= f[j];
        }
    }

    // 输出每个点的可达节点数
    for(int i=1;i<=n;i++){
        cout<<f[i].count()<<endl;
    }
}
```
