# 题目

https://www.acwing.com/problem/content/1136/

给出一个 N 个顶点 M 条边的无向无权图，顶点编号为 1 到 N。

问从顶点 1 开始，到其他每个点的最短路有几条。

#### 输入格式

第一行包含 2 个正整数 N,M，为图的顶点数与边数。

接下来 M 行，每行两个正整数 x,y，表示有一条顶点 x 连向顶点 y 的边，请注意可能有自环与重边。

#### 输出格式

输出 N 行，每行一个非负整数，第 i 行输出从顶点 1 到顶点 i 有多少条不同的最短路，由于答案有可能会很大，你只需要输出对 100003 取模后的结果即可。

如果无法到达顶点 i 则输出 0。

#### 数据范围

1≤N≤$10^5$,  
1≤M≤$2×10^5$

#### 输入样例：

```
5 7
1 2
1 3
2 4
3 4
2 3
4 5
4 5
```

#### 输出样例：

```
1
1
1
2
4
```

# 思路

这份代码解决的是经典图论问题：**求无权图（或边权均为 1 的图）的最短路路径条数**。

它在普通 **BFS（广度优先搜索）** 的基础上增加了一个计数逻辑，用来统计到达每个节点有多少种不同的走法。

---

### 1. 核心原理：最短路计数

在边权为 1 的图中，BFS 搜索天然保证了：**第一次搜到某个点时，走过的边数一定是最少的。**

为了统计路径数，我们引入 `cnt[j]` 数组：

- **新路径更短**：如果通过点 `t` 到达点 `j` 的距离比之前记录的更短，说明找到了更好的走法。此时，到达 `j` 的路径数就直接继承自 `t` 的路径数。
    
- **路径一样长**：如果通过点 `t` 到达点 `j` 的距离恰好等于已有的最短距离，说明发现了一条**新的、同样优的最短路**。此时，将 `t` 的路径数累加到 `j` 的路径数上。
    

---

### 2. 代码逻辑拆解

#### A. 初始化与起点

C++

```
memset(dist, 0x3f, sizeof dist); // 距离初始化为无穷大
dist[1] = 0; // 起点到自己的距离是 0
cnt[1] = 1;  // 起点到自己的最短路径只有 1 条
```

#### B. BFS 搜索过程

C++

```
while (hh <= tt)
{
    int t = q[hh ++ ]; // 取出队头元素

    for (int i = h[t]; ~i; i = ne[i]) // 遍历 t 的邻居 j
    {
        int j = e[i];
        
        // 情况 1：第一次发现 j，或者找到了更短的路
        if (dist[j] > dist[t] + 1)
        {
            dist[j] = dist[t] + 1; // 更新最短距离
            cnt[j] = cnt[t];       // 路径数继承自前驱点 t
            q[ ++ tt] = j;         // 入队，继续搜索
        }
        // 情况 2：找到了另一条一样长的最短路
        else if (dist[j] == dist[t] + 1)
        {
            cnt[j] = (cnt[j] + cnt[t]) % mod; // 累加路径数并取模
        }
    }
}
```

---

### 3. 为什么 BFS 可以这样做？

这个问题非常有深度。能够使用 BFS 计数的关键前提是：**图的拓扑顺序是按照距离层层递进的。**

在 BFS 中，点的访问顺序是按 `dist = 0, 1, 2, 3...` 依次进行的。

这意味着当我们要更新 `dist[j]` 的路径数时，能更新它的那些前驱点 `t`（满足 `dist[j] = dist[t] + 1`）一定已经先于 `j` 被处理完了。

---

### 4. 举个例子模拟一下

假设有 3 个点，关系是：`1-2, 1-3, 2-3`。

1. **处理点 1**：
    
    - 看点 2：`dist[2] = 1`, `cnt[2] = cnt[1] = 1`。
        
    - 看点 3：`dist[3] = 1`, `cnt[3] = cnt[1] = 1`。
        
2. **处理点 2**：
    
    - 看点 3：此时 `dist[3]` 已经是 1 了，而 `dist[2] + 1 = 2`。
        
    - 因为 $2 > 1$，所以这条路径不是最短路，忽略。
        
3. **结果**：
    
    - 到点 2 的最短路 1 条 (1->2)。
        
    - 到点 3 的最短路 1 条 (1->3)。
        

如果增加一个点 4，连接 `2-4` 和 `3-4`：

1. **处理点 2 时**：`dist[4] = 2`, `cnt[4] = 1`。
    
2. **处理点 3 时**：发现 `dist[4] == dist[3] + 1` (都是 2)。
    
3. **更新**：`cnt[4] = cnt[4] + cnt[3] = 2`。说明到 4 有两条最短路：`1-2-4` 和 `1-3-4`。
    

---

### 总结：这道题教会了你什么？

1. **BFS 的层次性**：BFS 不仅能找最短距离，还能利用“先搜到的距离一定不长于后搜到的”这一特性进行递推。
    
2. **动态规划思想**：最短路径计数其实是图上的动态规划，$cnt[j] = \sum cnt[t]$，只不过转移的顺序是由 BFS 维护的。
    
3. **取模运算**：在路径数可能非常巨大时，每一步累加都要记得 `% mod`。
    

# 代码

```c++
#include <cstdio>
#include <cstring>
#include <iostream>
#include <algorithm>

using namespace std;

const int N = 100010, M = 400010, mod = 100003;

int n, m;
int h[N], e[M], ne[M], idx;
int dist[N], cnt[N];
int q[N];

void add(int a, int b)
{
    e[idx] = b, ne[idx] = h[a], h[a] = idx ++ ;
}

void bfs()
{
    memset(dist, 0x3f, sizeof dist);
    dist[1] = 0;
    cnt[1] = 1;

    int hh = 0, tt = 0;
    q[0] = 1;

    while (hh <= tt)
    {
        int t = q[hh ++ ];

        for (int i = h[t]; ~i; i = ne[i])
        {
            int j = e[i];
            if (dist[j] > dist[t] + 1)
            {
                dist[j] = dist[t] + 1;
                cnt[j] = cnt[t];
                q[ ++ tt] = j;
            }
            else if (dist[j] == dist[t] + 1)
            {
                cnt[j] = (cnt[j] + cnt[t]) % mod;
            }
        }
    }
}

int main()
{
    scanf("%d%d", &n, &m);
    memset(h, -1, sizeof h);

    while (m -- )
    {
        int a, b;
        scanf("%d%d", &a, &b);
        add(a, b), add(b, a);
    }

    bfs();

    for (int i = 1; i <= n; i ++ ) printf("%d\n", cnt[i]);

    return 0;
}
```
