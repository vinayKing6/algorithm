思路:

这是一个典型的“合并果子”或“最小体力耗费”问题的贪心算法思路，其核心是使用小根堆（优先队列）来高效地找到当前最小的两个元素进行合并。

结合代码实现，其核心思路可以概括如下：

1. **初始化小根堆**：将所有果子的数量（或待合并的堆的重量）插入到一个小根堆中。小根堆能保证堆顶元素始终是最小的。

2. **循环合并**：当堆中的元素数量大于1时，重复以下操作： 

- **取出最小两值**：从堆中取出（删除）堆顶的两个元素，它们就是当前最小的两个值。

- **计算与累加**：将这两个值相加，得到本次合并的体力耗费，并将该耗费累加到总耗费中。

- **新值入堆**：将相加得到的新值作为一个新的“果子堆”重新插入小根堆中。

1. **输出结果**：当循环结束，堆中只剩下一个元素时，这个元素就是所有果子合并成一堆的总重量，而累计的总体力耗费即为所求的最小体力耗费值。

这个算法之所以正确，是基于贪心思想：每次合并当前最小的两堆，可以保证局部最优，并最终导向全局最优解。使用小根堆使每次获取最小值的操作时间复杂度为  O(log n) ，从而高效地解决了问题。

代码:

```c++
#include<bits/stdc++.h>  // 包含所有标准库头文件
using namespace std;      // 使用标准命名空间
﻿

const int N=10010;        // 定义最大数组大小，表示最多有10010堆果子
int n, fruits[N];         // n:果子堆数，fruits:存储各堆重量的数组
﻿

int main() {
    cin>>n;               // 读入果子的堆数n
    for(int i=0;i<n;i++)  // 循环读入每堆果子的重量
        cin>>fruits[i];
    sort(fruits, fruits+n);  // 将果子重量从小到大排序（预先排序）
    
    int res=0;            // 用于记录总体力消耗值
    
    // 创建最小堆（优先队列）：队头始终是最小元素
    // greater<int> 表示使用"大于"比较器，即小根堆
    priority_queue<int, vector<int>, greater<int>> heap;
    
    // 将所有果堆的重量压入最小堆
    for(int i=0;i<n;i++)
        heap.push(fruits[i]);
    
    // 当堆中元素多于1个时，继续合并
    while(heap.size()>1) {
        int tmp=heap.top();  // 取出最轻的果堆
        heap.pop();         // 弹出最轻的果堆
        
        tmp += heap.top();  // 再取出（当前）最轻的果堆，与第一堆合并
        heap.pop();         // 弹出第二轻的果堆
        
        heap.push(tmp);     // 将合并后新的果堆重量压入堆中
        res += tmp;        // 累加本次合并的体力消耗
    }
    
    cout<<res<<endl;       // 输出最小的体力耗费值
    return 0;
}
```
