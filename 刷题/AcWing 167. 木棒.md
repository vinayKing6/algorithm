# 题目

https://www.acwing.com/problem/content/169/

乔治拿来一组等长的木棒，将它们随机地砍断，使得每一节木棍的长度都不超过 50 个长度单位。

然后他又想把这些木棍恢复到为裁截前的状态，但忘记了初始时有多少木棒以及木棒的初始长度。

请你设计一个程序，帮助乔治计算木棒的可能最小长度。

每一节木棍的长度都用大于零的整数表示。

#### 输入格式

输入包含多组数据，每组数据包括两行。

第一行是一个不超过 64 的整数，表示砍断之后共有多少节木棍。

第二行是截断以后，所得到的各节木棍的长度。

在最后一组数据之后，是一个零。

#### 输出格式

为每组数据，分别输出原始木棒的可能最小长度，每组数据占一行。

#### 数据范围

数据保证每一节木棍的长度均不大于 50。

#### 输入样例：

```
9
5 2 1 5 2 1 5 2 1
4
1 2 3 4
0
```

#### 输出样例：

```
6
5
```

# 思路

![[Pasted image 20260307163505.png]]

这也是一道非常经典的搜索题目——**小木棒 (Sticks)**。它的核心考察点在于**深度优先搜索 (DFS)** 以及极其苛刻的**剪枝优化**。如果没有这些剪枝，程序在面对 $N=64$ 的数据时会运行到天荒地老。

题目大意：有一些同样长的小木棒被砍断成了若干节，现在给出砍断后的所有小节长度，求原木棒可能的**最小**长度。

---

### 1. 全局变量与核心逻辑

C++

```
const int N = 70;
int n;          // 木棒小节的数量
int w[N];        // 每小节的长度
int sum, length; // sum: 所有小节总长度；length: 当前尝试枚举的原木棒长度
bool st[N];      // 标记数组，记录某小节是否已被使用
```

---

### 2. 核心搜索函数 `dfs`

这个 `dfs` 的目标是：尝试用现有的碎片拼出 `sum / length` 根长度为 `length` 的原木棒。

C++

```
// u: 当前拼好了几根原木棒
// cur: 当前这根原木棒已经拼了多长
// start: 接下来从哪个下标开始枚举碎片（避免重复组合）
bool dfs(int u, int cur, int start)
{
    // 递归出口：如果拼好的长度刚好等于总长度，说明全部拼完，返回成功
    if (u * length == sum) return true;
    
    // 如果当前这根拼够了长度，开始拼下一根（u+1），当前长度清零，下标从0开始找
    if (cur == length) return dfs(u + 1, 0, 0);

    // 从 start 开始枚举每一个小碎片
    for (int i = start; i < n; i ++ )
    {
        // 剪枝 1：如果已被使用，或者加上它会超过目标长度，跳过
        if (st[i] || cur + w[i] > length) continue;

        st[i] = true; // 尝试使用
        if (dfs(u, cur + w[i], i + 1)) return true;
        st[i] = false; // 回溯

        // --- 以下是该题最精髓的 3 个剪枝 ---

        // 剪枝 2：如果这是这根原木棒的第一节碎片，却无法拼成功，则直接判定此路不通。
        // 原理：如果第一节就失败了，换其他同样长度的碎片在后面也会失败（因为碎片是对称的）。
        if (!cur) return false;

        // 剪枝 3：如果加上这节碎片后刚好拼满了一根原木棒，但后续搜索失败了。
        // 原理：贪心思想，如果刚好填满都搜不出结果，换用更细小的碎片去填满这个坑只会更难搜。
        if (cur + w[i] == length) return false;

        // 剪枝 4：跳过所有长度相同的碎片。
        // 原理：既然长度为 w[i] 的碎片不行，那么后面一连串长度也是 w[i] 的碎片肯定也不行。
        int j = i;
        while (j < n && w[j] == w[i]) j ++ ;
        i = j - 1;
    }

    return false;
}
```

---

### 3. 主函数逻辑

C++

```
int main()
{
    while (cin >> n, n) // 多组数据读入，直到 n 为 0
    {
        memset(st, 0, sizeof st);
        sum = 0;
        int max_w = 0; // 记录最长的一个碎片的长度

        for (int i = 0; i < n; i ++ )
        {
            cin >> w[i];
            sum += w[i];
            max_w = max(max_w, w[i]);
        }

        // 优化：从大到小排序。
        // 原理：先尝试长的碎片，可以让搜索树的分支更少，更快触发失败剪枝。
        sort(w, w + n);
        reverse(w, w + n);

        // 枚举原木棒的可能长度 length
        // length 必然在 [max_w, sum] 之间，且必须能被 sum 整除
        length = max_w; 
        while (true)
        {
            if (sum % length == 0 && dfs(0, 0, 0))
            {
                cout << length << endl;
                break;
            }
            length ++ ;
        }
    }
    return 0;
}
```

---

### 💡 剪枝要点总结（必考点）

1. **优化搜索顺序**：将碎片**从大到小排序**。因为长的碎片灵活度低，如果它不合适，能更早地导致失败从而回溯。
    
2. **排除等效冗余**：
    
    - 通过 `start` 变量确保枚举碎片时是递增的，避免 `(碎片1, 碎片2)` 和 `(碎片2, 碎片1)` 的重复计算。
        
    - 跳过相同长度的碎片（`while` 循环）。
        
3. **空棒剪枝**：如果在一根新木棒的开头（`cur == 0`）尝试第一个碎片就失败了，那么这个碎片在整道题的任何地方都不可能成功，直接 `return false`。
    
4. **满棒剪枝**：如果刚好凑成一根木棒（`cur + w[i] == length`）导致后续失败，不用尝试用更小的木棒去替换这一根，直接回溯。
    

理解这两个剪枝（剪枝 2 和剪枝 3）是掌握这道题的核心，因为它们体现了**“对称性消除”**和**“贪心失败”**的思想。

我们可以画一张逻辑决策树来辅助理解。

---

### 1. 剪枝 2：`if (!cur) return false;`

**含义**：如果在拼凑一根新的原木棒的**第一节碎片**时就失败了，那么这一整根原木棒就不用再尝试下去了。

- **为什么会这样？**
    
    - 当 `!cur` 时，代表我们刚刚开始拼一根全新的原木棒。
        
    - 如果我们尝试了当前可选的最长/最优的碎片 $w_i$，结果后面所有可能的组合都拼不出木棒，这意味着什么？
        
    - 这意味着**碎片 $w_i$ 根本无法成为任何一根木棒的起始件**。既然它连起始都做不到，那它在后面做“补位件”也必然会导致失败（或者产生对称的重复失败）。
        
    - 所以，一旦此时失败，无需再尝试其他的碎片，直接结束本次 DFS 并回溯到上一层。
        

---

### 2. 剪枝 3：`if (cur + w[i] == length) return false;`

**含义**：如果当前这节碎片刚好填满了这根原木棒，但后续搜索失败了，那就直接退出。

- **为什么会这样？**
    
    - 试想你正在拼最后一块积木，刚好拼成了一根完整的木棒。如果拼完后发现整个盘面剩下碎片的组合方式**行不通**。
        
    - 这意味着：用 `w[i]` 这一块大碎片填这个坑是错的。
        
    - **那么，用更细小的碎片去拼满这个坑，结果会好吗？**
        
    - 绝不会。因为如果你用 `w[j] + w[k]` (更小的碎片) 替代了 `w[i]`，你不仅消耗了更多的碎片数量，还让后续的拼凑过程变得更复杂。
        
    - 既然“大块”的都填不平后面，用“小块”的去填只会增加分支、消耗资源，导致后续更早失败。所以这种情况直接 `return false`，剪掉所有试图用小碎片替换大碎片的路径。
        

---

### 总结对比

|**剪枝类型**|**关键逻辑**|**核心思想**|
|---|---|---|
|**剪枝 2 (首个)**|`if (!cur) return false;`|**对称性**：如果起始块失败，说明该碎片无法放入当前结构，后续无需再试。|
|**剪枝 3 (填满)**|`if (cur + w[i] == length) return false;`|**贪心/最优性**：若填满后最终失败，说明这根木棒的组合方式无解，无需尝试更复杂的拆分。|

### 一个极简的例子

假设你要拼长度为 10 的木棒，手头有碎片：`[6, 4, 3, 3, 3, 1]`。

1. 你拼第一根，选了 `6`，接下来尝试 `4`。如果 `6+4` 拼成 10，但后面剩下 `[3, 3, 3, 1]` 没法拼成剩下的长度，导致失败。
    
2. **剪枝 3** 会告诉你：别再试用 `3+3+3+1` 去凑 10 了，因为 `6+4` 这种“大碎片”组合都挂了，用更多小碎片只会更糟。
    
3. **剪枝 2** 会告诉你：如果你最开始选了 `6` 放在起始位却失败了，说明 `6` 在这个盘面里根本没法作为木棒的起点，后续所有以 `6` 开头的方案全都是浪费时间。
    

**这两个剪枝之所以强大，是因为它们在搜索树还没长得太茂盛时，就直接砍掉了最底层的无效分支。**

这种“如果这一步选最优解都失败，那这一层就没必要选了”的逻辑，是否让你觉得搜索变得“聪明”了一些？如果还需要针对某个具体的 `start` 变量的作用进行解释，我可以再为你演示。

# 代码

```c++
#include<bits/stdc++.h>
using namespace std;

const int N = 70;

int w[N];       // 每根小木棒的长度
int n;          // 小木棒数量
int st[N];      // 标记某根木棒是否已经被使用
int length;     // 当前假设的原始木棒长度
int sum;        // 所有小木棒长度之和


// u：当前正在拼第几根原始木棒
// cur：当前这根木棒已经拼出来的长度
// start：枚举小木棒的起始位置（防止重复搜索）
bool dfs(int u,int cur,int start)
{
    // 如果已经拼出的总长度等于所有木棒长度
    // 说明所有木棒都已经成功拼完
    if(u * length == sum) return true;
    
    // 如果当前木棒已经拼到目标长度
    // 开始拼下一根木棒
    if(cur == length) return dfs(u+1,0,0);
    
    // 枚举可以选择的小木棒
    for(int i=start;i<n;i++)
    {
        // 如果该木棒已经用过
        if(st[i]) continue;
        
        // 如果加入这根木棒会超过目标长度
        if(cur + w[i] > length) continue;
        
        // 选择这根木棒
        st[i] = 1;
        
        // 递归继续拼当前木棒
        if(dfs(u,cur + w[i],i+1)) return true;
        
        // 回溯
        st[i] = 0;
        
        // 剪枝1：
        // 如果当前是新木棒的第一根都失败
        // 说明这种长度不可能拼出来
        if(!cur) return false;
        
        // 剪枝2：
        // 如果当前木棒刚好拼满但失败
        // 说明这种拼法不可能
        if(cur + w[i] == length) return false;
        
        // 剪枝3：
        // 跳过所有和当前木棒长度相同的木棒
        // 避免重复搜索
        int j = i;
        while(j < n && w[j] == w[i]) j++;
        i = j - 1;
    }
    
    return false;
}


int main()
{
    // 多组输入
    while(cin>>n,n)
    {
        memset(st,0,sizeof st);
        
        sum = 0;
        int max_len = 0;
        
        // 读入所有小木棒
        for(int i=0;i<n;i++)
        {
            cin>>w[i];
            sum += w[i];
            max_len = max(max_len,w[i]);
        }
        
        // 将木棒按长度从大到小排序
        // 大木棒优先放可以大幅减少搜索分支
        sort(w,w+n);
        reverse(w,w+n);
        
        // 枚举可能的原始木棒长度
        // 最小不小于最长的小木棒
        length = max_len;
        
        for(;length<=sum;length++)
        {
            // 必须能整除，否则不可能拼成
            if((sum % length) == 0 && dfs(0,0,0))
            {
                // 找到最小可行长度
                cout<<length<<endl;
                break;
            }
        }
    }
}
```
