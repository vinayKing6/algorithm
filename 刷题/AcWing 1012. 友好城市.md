# 题目

https://www.acwing.com/problem/content/1014/

Palmia国有一条横贯东西的大河，河有笔直的南北两岸，岸上各有位置各不相同的N个城市。

北岸的每个城市有且仅有一个友好城市在南岸，而且不同城市的友好城市不相同。

每对友好城市都向政府申请在河上开辟一条直线航道连接两个城市，但是由于河上雾太大，政府决定避免任意两条航道交叉，以避免事故。

编程帮助政府做出一些批准和拒绝申请的决定，使得在保证任意两条航线不相交的情况下，被批准的申请尽量多。

#### 输入格式

第1行，一个整数N，表示城市数。

第2行到第n+1行，每行两个整数，中间用1个空格隔开，分别表示南岸和北岸的一对友好城市的坐标。

#### 输出格式

仅一行，输出一个整数，表示政府所能批准的最多申请数。

#### 数据范围

1≤N≤5000,  
0≤xi≤10000

#### 输入样例：

```
7
22 4
2 6
10 3
15 12
9 8
17 17
4 2
```

#### 输出样例：

```
4
```

# 思路
这道题。。。。纯阅读理解啊
本题的背景是一个造桥项目，初始给定我们 n座打算建造的桥，每座桥有两个参数 x1 和 x2，表示该桥一头连接在上岸坐标为x1的地方，一头连在下岸坐标为x2的地方，题目要求我们找出一种建桥方案，使得在所有建造的桥不相交的前提下，建造尽可能多的桥，求出该方案的造桥数量
![[Pasted image 20260506145403.png]]
红色虚线表示初始提供的打算建造的桥

我们要找出的方案是在不相交的前提下的最大建桥数量

也就是上面的绿线连接而成的方案

这就是本题的大致意思

于是，可以轻易发现，上坐标从小到大枚举选择到的桥，其对应下坐标也必然是从小到大的

具体见下图：

蓝色表示该方案按照上坐标从小到大先选出来的桥

红色表示该方案的下一座桥的下坐标不是从小到大的，绿色表示是从小到大的

![[Pasted image 20260506150256.png]]

因此，该方案中，在上坐标排序的情况下，下坐标次序不是从小到大的，则必然不合法（会有相交）

于是，这题就变成了：桥以上坐标从小到大排序后，找出下坐标的最长上升子序列长度

这里贪心怎么证明是正确的？

我们要证明的是：**最大不相交子集的大小 = 排序后下坐标序列的 LIS 长度。**

- **可行性：** 既然 LIS 选出的序列在下坐标上是严格递增的，而我们预先对上坐标做了升序排列，那么根据上面的判定条件，LIS 选出的每一对桥之间都绝对不会相交。所以，LIS 的长度 $L$ 至少是一个合法的建桥数量。
    
- **最优性（反证）：** 假设存在一种更优的方案，能选出 $K$ 座桥且 $K > L$。
    
    根据“不相交”的物理约束，这 $K$ 座桥如果按上坐标排序，其下坐标也**必须**是递增的。但这与 “$L$ 是下坐标序列的最长上升子序列”这一前提矛盾。因为如果存在一个长度为 $K$ 的递增序列，LIS 的长度至少应该是 $K$。

# 代码

```c++
#include<bits/stdc++.h> // 包含 C++ 所有的标准库
using namespace std;

const int N = 5010;     // 根据题目数据范围设定数组大小
int n, f[N];            // n: 桥的数量, f[i]: 以第 i 座桥结尾的最长不相交序列长度
typedef pair<int, int> PII; // 使用 pair 存储桥的两个坐标，first 为上岸，second 为下岸
PII h[N];

int main() {
    // 输入桥的数量
    cin >> n;
    
    for(int i = 1; i <= n; i++) {
        f[i] = 1; // 初始化：每座桥自身就是一个长度为 1 的合法方案
        // 输入每座桥的上岸坐标(first)和下岸坐标(second)
        cin >> h[i].first >> h[i].second;
    }

    /**
     * 核心步骤 1：排序
     * 按照上岸坐标(first)从小到大进行排序。
     * 这样做是为了固定一个维度的顺序，使得我们只需要考虑下岸坐标(second)是否递增。
     * sort 是左闭右开区间，所以是 h+1 到 h+n+1。
     */
    sort(h + 1, h + n + 1);

    /**
     * 核心步骤 2：动态规划求解最长上升子序列 (LIS)
     * 在上岸坐标已经有序的前提下，如果下岸坐标也是递增的，这两座桥就不会相交。
     */
    for(int i = 1; i <= n; i++) {
        for(int j = 1; j < i; j++) {
            // 如果第 j 座桥的下岸坐标小于第 i 座桥的下岸坐标
            // 说明第 i 座桥可以接在第 j 座桥后面，且不相交
            if(h[j].second < h[i].second) {
                f[i] = max(f[i], f[j] + 1); // 尝试更新以 i 结尾的最长长度
            }
        }
    }

    // 统计所有 f[i] 中的最大值，即为全局最优解
    int res = 0;
    for(int i = 1; i <= n; i++) res = max(res, f[i]);
    
    cout << res << endl;
    
    return 0;
}
```
