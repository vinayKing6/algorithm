# 思路

这段代码是对你之前 DFS 逻辑的**动态规划（Dynamic Programming）**实现。它通过“递推”代替了“递归”，不仅解决了你之前代码中段数控制不准的问题，还显著提高了运行效率（尤其是避免了重复计算）。

其核心思想是：**将大问题拆解为“前 $p$ 个数分 $j-1$ 段”和“最后一段（从 $p+1$ 到 $i$）”两个子问题。**

---

### 1. 核心变量定义

- `prefix[i]`：**前缀和**。记录前 $i$ 个数字的和。目的是为了在 $O(1)$ 时间内计算出任意区间 $[p+1, i]$ 的和。
    
    - 公式：$Sum(p+1, i) = prefix[i] - prefix[p]$
        
- `dp[i][j]`：**状态表示**。
    
    - 含义：将序列的前 $i$ 个元素划分为 $j$ 段时，各段和的最大值之中的**最小值**。
        
    - 目标：求 `dp[n][k]`。
        

---

### 2. 状态转移方程

代码中的三层循环实现了以下转移逻辑：

$$dp[i][j] = \min_{j-1 \le p < i} \{ \max(dp[p][j-1], \text{abs}(prefix[i] - prefix[p])) \}$$

**通俗解释：**

如果你要把前 $i$ 个数分成 $j$ 段，你可以枚举最后一个分断点 $p$：

1. 假设前 $j-1$ 段负责处理前 $p$ 个数，其最优解为 `dp[p][j-1]`。
    
2. 那么最后一段（第 $j$ 段）就包含了从 $p+1$ 到 $i$ 的所有数，其和为 `current_abs`。
    
3. 这两部分合并后的最大值就是 `max(dp[p][j-1], current_abs)`。
    
4. 我们要遍历所有可能的 $p$，找到能让这个最大值最小的方案。
    

---

### 3. 代码逻辑详解

#### 初始化

C++

```
vector<vector<int>> dp(n+1, vector<int>(k+1, INF));
dp[0][0] = 0; 
```

- 将所有状态初始化为正无穷（`INF`），因为我们要找的是最小值。
    
- `dp[0][0] = 0` 表示 0 个元素分 0 段的和为 0，这是递推的起点。
    

#### 三层循环的作用

1. **第一层 (`i`)**：遍历元素数量，从 1 个数一直算到 $n$ 个数。
    
2. **第二层 (`j`)**：遍历分段数，从 1 段一直算到 $k$ 段。
    
3. **第三层 (`p`)**：寻找最优的分割点。$p$ 是前 $j-1$ 段处理到的位置。
    

#### 关键判断

C++

```
if(dp[p][j-1] == INF) continue;
```

这行代码确保了“前 $p$ 个数能分成 $j-1$ 段”这个前提是成立的（虽然在 $j \le i$ 的逻辑下通常成立，但这能增加代码鲁棒性）。

---

### 4. 算法复杂度分析

- **时间复杂度**：$O(n^2 \cdot k)$。
    
    - 如果 $n=100, k=100$，运算次数大约是 $10^6$，这在 C++ 环境下（通常 $10^8$ 次/秒）运行非常快。
        
- **空间复杂度**：$O(n \cdot k)$。
    
    - 使用了一个二维数组记录中间状态。
        

### 总结

相比于你之前的 DFS 做法，这个 DP 做法：

1. **更稳健**：严格控制了段数 $j$，不会出现漏算或逻辑死循环。
    
2. **更快速**：通过存储子问题的解，避免了 DFS 中对同一子区间的重复计算。
