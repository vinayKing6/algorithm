# 题目

[数独](https://baike.baidu.com/item/%E6%95%B0%E7%8B%AC/74847?fr=aladdin) 是一种传统益智游戏，你需要把一个 9×9 的数独补充完整，使得数独中每行、每列、每个 3×3 的九宫格内数字 1∼9 均恰好出现一次。

请编写一个程序填写数独。

#### 输入格式

输入包含多组测试用例。

每个测试用例占一行，包含 81 个字符，代表数独的 81 个格内数据（顺序总体由上到下，同行由左到右）。

每个字符都是一个数字（1−9）或一个 `.`（表示尚未填充）。

您可以假设输入中的每个谜题都只有一个解决方案。

文件结尾处为包含单词 `end` 的单行，表示输入结束。

#### 输出格式

每个测试用例，输出一行数据，代表填充完全后的数独。

#### 输入样例：

```
4.....8.5.3..........7......2.....6.....8.4......1.......6.3.7.5..2.....1.4......
......52..8.4......3...9...5.1...6..2..7........3.....6...1..........7.4.......3.
```

# 思路

练习: [[AcWing 843. n-皇后问题]]

搜索不用说,相信你一眼就可以看到是搜索算法,问题是这道题目纯搜索明显是要时间爆炸的,所以我们得剪枝.

优化搜索顺序:很明显,我们肯定是从当前能填合法数字最少的位置开始填数字

排除等效冗余:任意一个状态下,我们只需要找一个位置填数即可,而不是找所有的位置和可填的数字.

位运算:很明显这里面check判定很多,我们必须优化这个check,所以我们可以对于,每一行,每一列,每一个九宫格,都利用一个九位二进制数保存,当前还有哪些数字可以填写.

lowbit:我们这道题目当前得需要用lowbit运算取出当前可以能填的数字.

这段代码是 **数独（Sudoku）** 的高效搜索算法实现。它在经典的深度优先搜索（DFS）基础上，使用了 **位运算（Bitmask）** 进行状态压缩优化，并采用了 **启发式搜索（剪枝）** 策略。

## 核心原理：位运算优化

在传统的数独搜索中，我们需要用数组记录每行、每列、每个九宫格哪些数字被填过。这里改用一个 **9 位的二进制数**：

- 初始为 `(111111111)_2`，表示 1-9 都可以填。
    
- 如果数字 `i` 被填，则将第 `i` 位设为 `0`。
    
- 只有当行、列、九宫格对应的二进制位**同时为 1** 时，该数字才能填入。

## 逐行详细解释

### 1. 全局变量与初始化

C++

```
const int N = 9, M = 1 << N; // N为数独边长，M为二进制状态数（512）

int ones[M]; // 预处理：ones[i] 存储二进制数 i 中 1 的个数
int map[M];  // 预处理：map[1 << i] = i，快速通过 bit 找到对应的数字下标
int row[N], col[N], cell[3][3]; // 分别记录每行、列、九宫格可填数字的状态
char str[100]; // 存储 81 个字符的数独题目字符串
```

### 2. 初始化函数 `init()`

C++

```
void init()
{
    // 将所有行、列、九宫格的状态初始化为 511（二进制 9 个 1），表示全都可以填
    for (int i = 0; i < N; i ++ )
        row[i] = col[i] = (1 << N) - 1;

    for (int i = 0; i < 3; i ++ )
        for (int j = 0; j < 3; j ++ )
            cell[i][j] = (1 << N) - 1;
}
```

### 3. 更新状态 `draw()`

此函数用于在位置 $(x, y)$ 填写或擦除数字 $t$。

C++

```
void draw(int x, int y, int t, bool is_set)
{
    if (is_set) str[x * N + y] = '1' + t; // 填数
    else str[x * N + y] = '.';           // 回溯，擦除

    int v = 1 << t; // 数字 t 对应的二进制位
    if (!is_set) v = -v; // 如果是擦除，则加回这个值，否则减去

    // 更新行、列、九宫格的可用状态
    row[x] -= v;
    col[y] -= v;
    cell[x / 3][y / 3] -= v;
}
```

### 4. 获取可选数字 `get()`

C++

```
int get(int x, int y)
{
    // 利用按位与 (&)，只有三个集合都为 1 的位才是当前位置真正能填的数字
    return row[x] & col[y] & cell[x / 3][y / 3];
}

int lowbit(int x) { return x & -x; } // 返回二进制中最低位的 1 对应的数值
```

### 5. 核心搜索 `dfs()`

C++

```
bool dfs(int cnt) // cnt 为当前剩余空格数
{
    if (!cnt) return true; // 空格填完，找到答案

    // 【优化：启发式搜索】
    // 找出当前“可选数字最少”的格子优先填充，这能极大地缩小搜索树
    int minv = 10;
    int x, y;
    for (int i = 0; i < N; i ++ )
        for (int j = 0; j < N; j ++ )
            if (str[i * N + j] == '.')
            {
                int state = get(i, j);
                if (ones[state] < minv)
                {
                    minv = ones[state];
                    x = i, y = j;
                }
            }

    int state = get(x, y); // 获取该格子能填的所有数字
    // 遍历 state 中所有的 1（即可填数字）
    for (int i = state; i; i -= lowbit(i))
    {
        int t = map[lowbit(i)]; // 将最低位的 1 转换为数字 0-8
        draw(x, y, t, true);    // 尝试填数
        if (dfs(cnt - 1)) return true; // 递归
        draw(x, y, t, false);   // 回溯
    }

    return false;
}
```

### 6. 主函数 `main()`

C++

```
int main()
{
    // 预处理 map 和 ones，加速 DFS 运行
    for (int i = 0; i < N; i ++ ) map[1 << i] = i;
    for (int i = 0; i < 1 << N; i ++ )
        for (int j = 0; j < N; j ++ )
            ones[i] += i >> j & 1;

    while (cin >> str, str[0] != 'e') // 输入 'end' 时结束
    {
        init();
        int cnt = 0;
        for (int i = 0, k = 0; i < N; i ++ )
            for (int j = 0; j < N; j ++, k ++ )
                if (str[k] != '.') {
                    int t = str[k] - '1';
                    draw(i, j, t, true); // 预填题目给出的数字
                }
                else cnt ++ ; // 统计空格数

        dfs(cnt);
        puts(str); // 输出填完后的结果串
    }
    return 0;
}
```

## 总结：为什么这个代码快？

1. **位运算加速**：使用 `&` 运算代替繁琐的循环检查，判断“能不能填”只需一步。
    
2. **Lowbit 技术**：利用 `lowbit` 快速跳过无效位，只处理可填的数字。
    
3. **最少选项优先（MRV 启发式）**：这是数独算法的核心，优先填选择最少的格子，就像我们平时做数独先填那些“显而易见”的格子一样，这能剪掉绝大部分无效搜索分支。
