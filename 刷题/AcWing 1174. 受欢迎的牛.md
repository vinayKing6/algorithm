# 题目

https://www.acwing.com/problem/content/1176/

每一头牛的愿望就是变成一头最受欢迎的牛。

现在有 N 头牛，编号从 1 到 N，给你 M 对整数 (A,B)，表示牛 A 认为牛 B 受欢迎。

这种关系是具有传递性的，如果 A 认为 B 受欢迎，B 认为 C 受欢迎，那么牛 A 也认为牛 C 受欢迎。

你的任务是求出有多少头牛被除自己之外的所有牛认为是受欢迎的。

#### 输入格式

第一行两个数 N,M；

接下来 M 行，每行两个数 A,B，意思是 A 认为 B 是受欢迎的（给出的信息有可能重复，即有可能出现多个 A,B）。

#### 输出格式

输出被除自己之外的所有牛认为是受欢迎的牛的数量。

#### 数据范围

1≤N≤$10^4$,  
1≤M≤$5×10^4$

#### 输入样例：

```
3 3
1 2
2 1
2 3
```

#### 输出样例：

```
1
```

#### 样例解释

只有第三头牛被除自己之外的所有牛认为是受欢迎的。

# 思路
先建图然后用dfs一个一个判断能连通哪些牛,一条有向边代表A受B欢迎 很显然会TLE,过了19/24个样例

下面这个tarjan算法太难了太抽象了,机试的话直接dfs吧!

这份代码解决的是经典的有向图 **强连通分量（SCC, Strongly Connected Components）** 问题。通常对应题目《受欢迎的牛》。

题目大意是：有 $n$ 头牛，牛之间的“欢迎”具有传递性（A 欢迎 B，B 欢迎 C，则 A 也欢迎 C）。如果一头牛被所有其他的牛欢迎，它就是“明星”。求明星牛的数量。

---

### 第一步：理解核心——强连通分量 (SCC)

在一个有向图中，如果两个点 $A$ 和 $B$ 能够 **互相到达**，我们就说它们是强连通的。

**Tarjan 算法** 的作用就是把图中所有的“环”找出来，把每个环看作一个整体（缩点）。

- **为什么要缩点？** 因为在一个环里的所有牛，由于欢迎具有传递性，它们互相之间都是欢迎的。如果环外有一头牛欢迎环内的某一个成员，那它就欢迎环内的所有人。
    

---

### 第二步：Tarjan 算法核心变量

代码里的这些数组是算法的灵魂：

1. `dfn[u]`：**时间戳**。记录节点 $u$ 是第几个被搜到的。
    
2. `low[u]`：**回溯值**。记录 $u$ 所在的连通分量能追溯到的最早的节点时间戳。
    
3. `stk[]` 和 `in_stk[]`：辅助栈，用来暂时保存当前搜索路径上的点。
    
4. `id[u]`：缩点后，节点 $u$ 属于第几个强连通分量。
    
5. `Size[i]`：第 $i$ 个强连通分量里包含原图节点的个数。
    

---

### 第三步：代码逻辑拆解

#### 1. Tarjan 函数：找环与缩点

C++

```
void tarjan(int u){
    dfn[u] = low[u] = ++ timestamp; // 初始化时间戳
    stk[ ++ top] = u, in_stk[u] = true; // 入栈
    
    for (int i = h[u]; i != -1; i = ne[i]) {
        int j = e[i];
        if (!dfn[j]) { // 如果 j 没走过
            tarjan(j);
            low[u] = min(low[u], low[j]); // 用子节点更新回溯值
        }
        else if (in_stk[j]) // 如果 j 在栈中，说明搜到了一个环
            low[u] = min(low[u], dfn[j]);
    }

    if (dfn[u] == low[u]) { // 如果回到了分量的根部
        ++ scc_cnt; // 发现一个新的 SCC
        int y;
        do {
            y = stk[top -- ];
            in_stk[y] = false;
            id[y] = scc_cnt; // 标记节点所属的 SCC 编号
            Size[scc_cnt] ++ ; // 统计这个分量里有多少点
        } while (y != u);
    }
}
```

这部分是 Tarjan 算法的核心，它利用 **DFS（深度优先遍历）** 的性质来判定“环”。

要理解这段代码，必须先搞清楚 `dfn` 和 `low` 的直观含义：

- **`dfn[u]`**：你是第几个被点名的（身份编号，固定不变）。
    
- **`low[u]`**：你**通过眼前的边**或者**你的子孙后代**，能跳回到的“编号最小”的祖先是谁（最高理想，不断更新）。
    

---

##### 模拟情景：寻找隐藏的“环”

假设有三头牛：**1 → 2，2 → 3，3 → 1**。这是一个典型的环。

###### 1. 第一步：从 1 号出发

- 执行 `tarjan(1)`。
    
- `dfn[1] = low[1] = 1`。
    
- 把 **1** 压入栈。
    

###### 2. 第二步：1 走到 2

- 发现 `dfn[2]` 是 0（没走过），执行 `tarjan(2)`。
    
- `dfn[2] = low[2] = 2`。
    
- 把 **2** 压入栈。
    

###### 3. 第三步：2 走到 3

- 发现 `dfn[3]` 是 0（没走过），执行 `tarjan(3)`。
    
- `dfn[3] = low[3] = 3`。
    
- 把 **3** 压入栈。
    

###### 4. 第四步：3 发现了一条通往 1 的边（关键！）

- 此时看 3 的邻接点 `j = 1`。
    
- `dfn[1]` 已经是 1 了（走过了），进入 `else if (in_stk[1])` 分支。
    
- **3 号的心态**：“喔！我发现 1 号还在栈里，说明 1 号是我的祖先，我们成环了！”
    
- 更新 `low[3] = min(low[3], dfn[1])`。此时 `low[3]` 变成了 **1**。
    

###### 5. 第五步：递归回溯（更新整个环）

- `tarjan(3)` 结束，回到 `tarjan(2)`。
    
- 2 号收到 3 号的反馈：`low[2] = min(low[2], low[3])`。
    
- **2 号的心态**：“既然 3 号能跳回 1 号，那我也能通过 3 号跳回 1 号。”
    
- 更新 `low[2]` 变成了 **1**。
    
- `tarjan(2)` 结束，回到 `tarjan(1)`。
    
- 1 号收到 2 号的反馈：`low[1] = min(low[1], low[2])`。
    
- `low[1]` 保持为 **1**。
    

---

##### 为什么判断 `else if (in_stk[j])`？

这是初学者最容易晕的地方。为什么要判断 **“在栈中”**？

在有向图 DFS 中，你会遇到两种“已经走过”的点：

1. **在栈里的点**：它是你的**祖先**。如果你能连到它，说明你们之间存在一条回访边，**一定有环**。
    
2. **不在栈里的点**：它已经被搜完了，且被判定属于**另一个**已经封闭的强连通分量。你连向它，并不能构成环。
    

---

##### 最后的判别式：`if (dfn[u] == low[u])`

当回溯结束时，如果一个点的“身份编号”等于它的“最高理想”：

- 说明 $u$ 是这个环的**最高点**（所有在这个环里的点，最终 `low` 都会指向 $u$ 的 `dfn`）。
    
- 这时候就把栈里从 $u$ 往上的所有点全部弹出，它们共同组成一个 **SCC**。
    

**用一句话总结：**

`dfn` 是入队时的座次，`low` 是通过环能摸到的最高座位；当回溯发现你的 `low` 没变小，说明你就是这个圈子的头儿，该带大家“出栈”自立门户了。

这个模拟过程让你对 `low` 的变化有感觉了吗？需要我再对比一下 **“无向图”** 中的处理方式吗（那里不能用 `dfn[j]` 而是要看父节点）？

#### 2. 统计出度：逻辑转换

缩点之后，原图变成了一个 **有向无环图 (DAG)**。

- **出度 (`dout`)**：如果一个强连通分量 $A$ 有出边连向 $B$，说明 $A$ 里的牛欢迎 $B$ 里的牛，但 $B$ 不一定欢迎 $A$。
    
- **明星的特征**：在缩点后的图中，如果一个分量的 **出度为 0**，说明没有它欢迎的人。
    

C++

```
for (int i = 1; i <= n; i ++ )
    for (int j = h[i]; ~j; j = ne[j]) {
        int k = e[j];
        int a = id[i], b = id[k];
        if (a != b) dout[a] ++ ; // 只要 A 分量有指向 B 分量的边，A 的出度就 +1
    }
```

#### 3. 寻找唯一的终点

- 如果图中 **只有一个** 出度为 0 的分量，那么这个分量里的所有牛都被其他所有牛欢迎，答案就是 `Size[i]`。
    
- 如果图中 **有多个** 出度为 0 的分量，说明这些分量之间互不欢迎，那就没有明星，答案是 0。
    

C++

```
int zeros = 0, sum = 0;
for (int i = 1; i <= scc_cnt; i ++ )
    if (!dout[i]) {
        zeros ++ ;
        sum += Size[i];
        if (zeros > 1) { // 发现多个出度为 0 的点，凉凉
            sum = 0;
            break;
        }
    }
```

## 为什么出度必须唯一才有解?
### 1. 核心大前提：欢迎具有“传递性”

如果 A 欢迎 B，B 欢迎 C，那么 A 就一定欢迎 C。

这意味着：**在缩点后的图中，只要有一条路径能从 A 走到 Z，那么 A 就欢迎 Z。**

---

### 2. 为什么“明星”一定在出度为 0 的块里？

想象有两个块（强连通分量）：**块 A** 和 **块 B**。

- 如果 **A 有一条边指向 B**，说明 A 里的牛欢迎 B。
    
- 但是 B 没有边指向 A（因为缩点后没有环了），所以 B **不欢迎** A。
    
- 既然有人不欢迎 A，那 A 里的牛肯定成不了“全场明星”。
    

**结论：** 只要一个块有“出路”，它就一定不是明星，因为被它指向的那些块不会回过头来欢迎它。**只有走到“关系的尽头”（出度为 0）的地方，才可能是明星。**

---

### 3. 为什么出度为 0 的块“必须唯一”？

我们来看一个**反例**。假设有两群牛（块 X 和 块 Y），它们的**出度都是 0**。

- **看块 X**：它是终点，没有路通往 Y。这意味着 X 里的牛**不欢迎** Y。
    
- **看块 Y**：它也是终点，没有路通往 X。这意味着 Y 里的牛**不欢迎** X。
    

**现在的尴尬局面：**

- X 想当明星？不行，Y 不欢迎它。
    
- Y 想当明星？不行，X 不欢迎它。
    
- **结果：** 全场没有明星。
    

**所以：** 一旦出现了两个“终点”，它们之间互不往来，就没有任何一个点能收集到全场所有的“欢迎”。

---

### 4. 为什么“唯一终点”就一定是全场明星？

如果全图**只有一个**出度为 0 的块 Z：

1. **连通性保证**：在有向无环图中，你从任何一个点出发，顺着边走，最终**一定会**走到某个出度为 0 的点。
    
2. **殊途同归**：因为终点只有一个（就是 Z），所以不管你从 A、B 还是 C 出发，最终都会汇聚到 Z。
    
3. **欢迎送达**：既然所有点都能走到 Z，根据传递性，所有点的牛都欢迎 Z。
    
4. **Z 内部互粉**：Z 是强连通分量，内部本来就互粉。
    
5. **结论**：Z 里的牛被全场欢迎，它们就是明星！
    

---

### 5. 举个简单的例子

假设有 4 头牛：

- 1 欢迎 2
    
- 3 欢迎 4
    
- 2 欢迎 4
    

**缩点后的图：** `1 -> 2 -> 4` 且 `3 -> 4`。

- **出度计算：**
    
    - 块 1：出度 1（指向 2）
        
    - 块 2：出度 1（指向 4）
        
    - 块 3：出度 1（指向 4）
        
    - **块 4：出度 0**
        
- **明星判定：** 唯一的出度为 0 的块是 {4}。
    
    - 1 欢迎 4 吗？欢迎（1->2->4）。
        
    - 2 欢迎 4 吗？欢迎（2->4）。
        
    - 3 欢迎 4 吗？欢迎（3->4）。
        
    - **结果：** 4 是明星。
        

**如果再加一条边：** `4 欢迎 1`

- 那么 1, 2, 4 构成一个大环，缩成一个大块 Z。
    
- 此时图变成：`3 -> Z`，且 `Z` 出度为 0。
    
- **结果：** Z 里的所有牛（1, 2, 4）都是明星。
    

**如果再加一头牛 5，且 5 谁也不欢迎：**

- 此时 **4** 的出度为 0，**5** 的出度也为 0。
    
- 4 不欢迎 5，5 也不欢迎 4。明星数量瞬间变成 0。
    

---

### 总结

- **出度 = 0**：代表它是“被欢迎”的终点站。
    
- **唯一性**：确保全场只有这一个“终点站”，所有路都通向这里。
    

这一套逻辑下来，是不是清晰很多了？想一想，如果我们要找的是“欢迎所有人的牛”，是不是对应的就是“入度为 0 且唯一”呢？

---

这个问题确实很微妙，我们不看复杂的代码，直接用**“染料”**和**“路径”**来做一次思维实验。

想象一下：有两个圆环，它们像数字 **“8”** 一样，中间重叠了一个点。
## 思维实验: 如果一个点同时属于两个环
### 1. 为什么它们会变成“一坨”？

强连通分量的核心定义是：**只要 A 能到 B，且 B 也能到 A，它们就是一体的。**

假设左边的环是 $\{1, 2, 3\}$，右边的环是 $\{3, 4, 5\}$，点 **3** 是公共点。

- **从 1 出发能到 5 吗？** 可以。路径：$1 \to 2 \to 3 \to 4 \to 5$。
    
- **从 5 出发能回 1 吗？** 可以。路径：$5 \to 3 \to 1$（或者 $5 \to 3 \to 2 \to 1$）。
    

既然 $1$ 和 $5$ 可以互通，那么根据定义，它们必须属于同一个强连通分量。这就是为什么 Tarjan 会把这两个环里所有的点（1, 2, 3, 4, 5）全部染成同一个 `id`。

---

### 2. Tarjan 在栈里是怎么处理这个“8”字形的？

我们跟着 DFS 的脚步走一遍，看 `low` 数组是如何“通气”的：

1. **进入左环：** DFS 从 $1$ 走到 $2$，再走到 $3$。此时栈：`[1, 2, 3]`。
    
2. **进入右环：** 在 $3$ 号点，DFS 决定先去探索右边的环，走到 $4$，再走到 $5$。此时栈：`[1, 2, 3, 4, 5]`。
    
3. **发现第一个环：** $5$ 号点有一条边连回 $3$。
    
    - $5$ 发现 $3$ 在栈里，于是 $low[5]$ 更新为 $dfn[3]$。
        
4. **回溯：** 递归回到 $4$，$low[4]$ 接收了 $5$ 的信息，也变成了 $dfn[3]$。
    
5. **回到交点：** 递归回到 $3$。此时 $3$ 还没完，它发现左环还有一条边连回 $1$。
    
    - $3$ 发现 $1$ 也在栈里，于是 $low[3]$ 更新为 $dfn[1]$。
        
6. **全员“降级”：** 重点来了！当递归继续往回走，回到 $4$ 和 $5$ 的逻辑（如果有的话）或者回到 $2$ 和 $1$ 时，**所有点的 `low` 最终都会被同步成最老的祖先 $dfn[1]$。**
    

---

### 3. 最终的“弹出”时刻

只有当 DFS 彻底回溯到 **1 号点**（这个大分量的根）时，Tarjan 才会检查发现：`dfn[1] == low[1]`。

此时，它会看一眼栈：`[1, 2, 3, 4, 5]`。

它会一口气把这 **5 个元素全部弹出**，并告诉它们：“你们都属于 `id = 1` 的大集体，这个集体的 `Size` 是 5。”

---

### 总结

你担心的“一个点属于两个环”会导致混乱，但在 Tarjan 看来：

- **点 3 是一个桥梁**：它把左边的能量传给右边，把右边的能量传回左边。
    
- **栈是一个大熔炉**：只要这些点还没被确定为“到此为止”，它们就一直在栈里待命。
    
- **缩点是一个粉碎机**：无论你有多少个环重叠，只要它们能互通，缩点后就只剩下一个**超级节点**。
    

**这个“合体”的过程，确保了我们在计算出度时，是把这“一坨”点当作一个整体来看待的。** 这样就不会出现“环 A 指向 环 B，环 B 又指向 环 A”导致出度算错的情况了。

明白了“合体”逻辑后，你是不是对 `low[u] = min(low[u], low[j])` 这行代码的强大有了更深的体会？它是真的能把整个大圈子的信息都“同步”起来。

你想再看看如果这两个环**不重叠**，只是有一条单向边相连（$A \to B$），Tarjan 的栈又会发生什么奇妙的变化吗？
## 总结：这道题教会了你什么？

1. **复杂关系的简化**：通过 Tarjan 缩点，把相互依赖的“环”简化成一个“点”。
    
2. **出度与入度的含义**：
    
    - 在有向无环图中，出度为 0 的点是“终点”。
        
    - 在“欢迎”这种具有传递性的关系中，终点代表被所有人指向。
        
3. **Tarjan 的固定模板**：`dfn` 和 `low` 的更新逻辑是处理所有强连通问题的万能钥匙。
    

# 代码

```c++
#include<bits/stdc++.h>
using namespace std;

// N为点数，M为边数
const int N=1e4+10, M=5e4+10;
int n, m, h[N], e[M], ne[M], idx;

// Tarjan 核心数组
int dfn[N];      // 时间戳：记录节点 i 是第几个被搜到的
int low[N];      // 回溯值：记录节点 i 能够跳回到的最早的祖先的时间戳
int timestamp;   // 全局时间戳计数器
int in_stk[N];   // 标记数组：判断节点是否在当前的搜索栈中
stack<int> s;    // 搜索栈：保存当前 SCC 的候选节点

// 缩点后使用的数组
int id[N];       // 记录节点 i 属于哪个强连通分量 (SCC)
int scc_cnt;     // 强连通分量的总数
int Size[N];     // 记录每个 SCC 中包含的原图节点个数
int dout[N];     // 记录每个 SCC 的出度

void add(int a, int b){
    e[idx] = b;
    ne[idx] = h[a];
    h[a] = idx++;
}

void tarjan(int u){
    // 初始化当前节点的时间戳和回溯值
    dfn[u] = low[u] = ++timestamp;
    s.push(u);       // 入栈
    in_stk[u] = 1;   // 标记已在栈中
    
    for(int i = h[u]; ~i; i = ne[i]){
        int j = e[i];
        if(!dfn[j]){ // 如果 j 没被访问过，说明是树枝边
            tarjan(j);
            // 回溯时更新 u 的回溯值：u 能跳到的地方，j 肯定也能跳到
            low[u] = min(low[u], low[j]);
        } else if(in_stk[j]){ 
            // 如果 j 已经在栈中，说明搜到了一个环（后向边）
            // 更新 u 能跳回到的最早祖先
            low[u] = min(low[u], dfn[j]);
        }
    }
    
    // 如果 dfn == low，说明 u 是当前强连通分量的“最高点”或“根”
    if(low[u] == dfn[u]){
        ++scc_cnt; // 发现一个新的 SCC
        int k;
        do {
            k = s.top();
            s.pop();
            in_stk[k] = 0;   // 出栈后取消标记
            id[k] = scc_cnt; // 记录节点所属的 SCC 编号
            Size[scc_cnt]++; // 累加该 SCC 的大小
        } while(k != u);     // 将栈中 u 之上的所有点全部弹出
    }
}

int main(){
    // 提高输入效率
    ios::sync_with_stdio(false); cin.tie(0);
    
    cin >> n >> m;
    memset(h, -1, sizeof h);
    while(m--){
        int a, b;
        cin >> a >> b;
        add(a, b);
    }
    
    // 图可能不连通，需要对每个未访问的点跑一遍 Tarjan
    for(int i = 1; i <= n; i++){
        if(!dfn[i]) tarjan(i);
    }
    
    // 遍历所有原图的边，统计缩点后 SCC 的出度
    for(int i = 1; i <= n; i++){
        for(int j = h[i]; ~j; j = ne[j]){
            int k = e[j];
            int a = id[i], b = id[k]; // 获取边两端点所属的 SCC 编号
            if(a != b) {
                dout[a]++; // 如果这条边连接两个不同的 SCC，则起始 SCC 的出度 +1
            }
        }
    }
    
    int zeros = 0, sum = 0;
    // 遍历所有 SCC，寻找出度为 0 的分量
    for(int i = 1; i <= scc_cnt; i++){
        if(!dout[i]){
            zeros++;      // 统计出度为 0 的 SCC 数量
            sum = Size[i]; // 暂时记录该 SCC 的大小
            // 如果发现多个 SCC 出度都为 0，说明它们互不欢迎，全图无明星
            if(zeros > 1){
                sum = 0;
                break;
            }
        }
    }
    
    cout << sum << endl;
    return 0;
}
```
