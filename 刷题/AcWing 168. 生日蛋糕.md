# 题目

https://www.acwing.com/problem/content/170/

7 月 17 日是 Mr.W 的生日，ACM-THU 为此要制作一个体积为 Nπ 的 M 层生日蛋糕，每层都是一个圆柱体。

设从下往上数第 i 层蛋糕是半径为 $R_i$，高度为 $H_i$ 的圆柱。

当 i<M 时，要求 $R_i>R_{i+1}$ 且 $H_i>H_{i+1}$。

由于要在蛋糕上抹奶油，为尽可能节约经费，我们希望蛋糕外表面（最下一层的下底面除外）的面积 Q 最小。

令 Q=Sπ ，请编程对给出的 N 和 M，找出蛋糕的制作方案（适当的 Ri 和 Hi 的值），使 S 最小。

除 Q 外，以上所有数据皆为正整数。

#### 输入格式

输入包含两行，第一行为整数 N，表示待制作的蛋糕的体积为 Nπ。

第二行为整数 M，表示蛋糕的层数为 M。

#### 输出格式

输出仅一行，是一个正整数 S（若无解则 S=0）。

#### 数据范围

1≤N≤10000,  
1≤M≤20

#### 输入样例：

```
100
2
```

#### 输出样例：

```
68
```

# 思路

这道题是经典的搜索优化题 **“生日蛋糕”**。题目要求在给定总体积 $N$ 和层数 $M$ 的情况下，设计一个圆柱体叠加的蛋糕，使得表面积 $S$ 最小。

代码的核心是 **DFS + 5个关键剪枝**。

---

### 1. 核心数学公式

- **体积**：$V = \sum_{i=1}^{M} \pi R_i^2 H_i$
    
- **表面积**：$S = \pi R_m^2 + \sum_{i=1}^{M} 2\pi R_i H_i$
    
    - 注：代码中为了计算方便，统一去掉了 $\pi$。
        
    - $R_m^2$ 是最底层蛋糕的顶面积，从上往下看，所有层的露出的环形面积之和刚好等于最底层的圆面积。
        

---

### 2. 预处理与全局变量

C++

```
int minv[N], mins[N]; // 预处理：从上往下数，前 i 层的最小体积和最小侧面积
int R[N], H[N];       // 记录每一层的半径和高度
int ans = INF;        // 最终答案（最小表面积）

// main 函数中的预处理
for (int i = 1; i <= m; i ++ ) {
    // 第 i 层（从上往下数）最小半径是 i, 最小高度是 i
    minv[i] = minv[i - 1] + i * i * i; 
    mins[i] = mins[i - 1] + 2 * i * i;
}
```

---

### 3. DFS 函数：核心剪枝

`dfs(u, v, s)` 表示正在搜第 `u` 层（从下往上数），当前已用体积 `v`，当前已用侧面积 `s`。

#### 剪枝 1：可行性剪枝

C++

```
if (v + minv[u] > n) return; 
```

如果当前体积 + 剩余层数的最小体积 > 总需求体积 $N$，直接撤退。

#### 剪枝 2：最优性剪枝

C++

```
if (s + mins[u] >= ans) return;
```

如果当前面积 + 剩余层数的最小侧面积 $\ge$ 已经搜到的最优解，直接撤退。

#### 剪枝 3：数学推导剪枝（最重要的剪枝）

C++

```
if (s + 2 * (n - v) / R[u + 1] >= ans) return;
```

- **原理**：利用剩余体积推导剩余侧面积的下界。
    
- 剩余侧面积 $S_{rem} = \sum 2 R_i H_i = \sum \frac{2 R_i^2 H_i}{R_i}$。
    
- 因为 $R_i < R_{u+1}$（下一层半径大于当前层），所以 $S_{rem} > \frac{2}{R_{u+1}} \sum R_i^2 H_i$。
    
- 即 $S_{rem} > \frac{2(N - V_{cur})}{R_{u+1}}$。如果这个理论最小值都比当前答案差，就剪枝。
    

---

### 4. 搜索顺序与枚举范围

C++

```
// 每一层必须比下一层小：R[u] < R[u+1], H[u] < H[u+1]
// R 的上限受两个限制：1. R[u+1]-1; 2. 剩余体积约束（假设 h 最小为 1）
for (int r = min(R[u + 1] - 1, (int)sqrt(n - v)); r >= u; r -- ) 
    for (int h = min(H[u + 1] - 1, (n - v) / r / r); h >= u; h -- )
    {
        int t = 0;
        if (u == m) t = r * r; // 如果是最底层，加上顶面积
        R[u] = r, H[u] = h;
        dfs(u - 1, v + r * r * h, s + 2 * r * h + t);
    }
```

- **搜索顺序优化**：`r` 和 `h` 从大到小枚举。因为底层越大，体积消耗越快，能更早触发剪枝。
    

---

### 5. 总结

这段代码体现了搜索题的终极奥义：

1. **确定方向**：从下往上搜（体积大，约束强）。
    
2. **范围确定**：用 $R_{u+1}$ 和剩余体积限制当前层的 $R, H$。
    
3. **剪枝加持**：
    
    - **可行性**（不能超体积）。
        
    - **最优性**（不能超面积）。
        
    - **下界预测**（利用公式 $S_{rem} > \frac{2 \times V_{rem}}{R}$）。
