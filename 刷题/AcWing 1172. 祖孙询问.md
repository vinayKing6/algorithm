# 题目

https://www.acwing.com/problem/content/1174/

给定一棵包含 n 个节点的有根无向树，节点编号互不相同，但不一定是 1∼n。

有 m 个询问，每个询问给出了一对节点的编号 x 和 y，询问 x 与 y 的祖孙关系。

#### 输入格式

输入第一行包括一个整数 表示节点个数；

接下来 n 行每行一对整数 a 和 b，表示 a 和 b 之间有一条无向边。如果 b 是 −1，那么 a 就是树的根；

第 n+2 行是一个整数 m 表示询问个数；

接下来 m 行，每行两个不同的正整数 x 和 y，表示一个询问。

#### 输出格式

对于每一个询问，若 x 是 y 的祖先则输出 1，若 y 是 x 的祖先则输出 2，否则输出 0。

#### 数据范围

1≤n,m≤$4×10^4$,  
1≤每个节点的编号≤$4×10^4$

#### 输入样例：

```
10
234 -1
12 234
13 234
14 234
15 234
16 234
17 234
18 234
19 234
233 19
5
234 233
233 12
233 13
233 15
233 19
```

#### 输出样例：

```
1
0
0
0
2
```

这道题是 **LCA（最近公共祖先）经典题**，使用的是 **倍增法 + BFS预处理**。  
我按 **理解 → 算法 → 代码结构 → 查询逻辑** 一步一步给你讲清楚。

## 一、题目本质

输入一棵树，每个节点给出：

```
a b
```

表示

```
a 的父节点是 b
```

如果

```
b = -1
```

说明：

```
a 是根节点
```

然后有 m 次查询：

```
a b
```

判断：

|输出|含义|
|---|---|
|1|a 是 b 的祖先|
|2|b 是 a 的祖先|
|0|都不是|

---

## 二、为什么要用 LCA

如果：

```
LCA(a,b) = a
```

说明：

```
a 是 b 的祖先
```

如果：

```
LCA(a,b) = b
```

说明：

```
b 是 a 的祖先
```

否则：

```
两者只是普通节点
```

### 加边函数

```cpp

{
    e[idx] = b;
    ne[idx] = h[a];
    h[a] = idx ++ ;

```

表示：

```

```

但树是 **无向图**：

```cpp

```

## 四、核心算法：倍增 LCA

思想：

每个节点记录

```
2^k 级祖先
```

例如：

```
fa[x][0] = 父节点
fa[x][1] = 2^1祖先
fa[x][2] = 2^2祖先
fa[x][3] = 2^3祖先
```

公式：

```
fa[x][k] = fa[ fa[x][k-1] ][k-1]
```

意思：

```
2^k祖先 = 2^(k-1)祖先的2^(k-1)祖先
```

---

## 五、为什么是 16

代码：

```cpp
fa[N][16]
```

因为：

```
2^15 = 32768
```

已经覆盖：

```
40000
```

所以：

```
log2(40000) ≈ 16
```

## 十一、LCA查询函数

```cpp

```

步骤：

```

```

## 十二、第一步：对齐深度

```cpp
if (depth[a] < depth[b]) swap(a, b);
```

保证：

```
a 更深
```

---

向上跳：

```cpp
for (int k = 15; k >= 0; k--)
    if (depth[fa[a][k]] >= depth[b])
        a = fa[a][k];
```

意思：

```
a 向上跳 2^k
直到和 b 同层
```

## 十九、最容易记忆的 LCA 模板

口诀：

```
先拉平
再一起跳
跳到刚好不同
父亲就是LCA
```

你问的是 **LCA代码里为什么要用 0 号点**，不是差分约束里的那个。这两个 **0号点的作用完全不同**。

在你给的 LCA 代码里，`0` 号点其实是 **一个虚拟节点（空节点）**，主要是为了 **防止数组越界和简化判断逻辑**。

我给你分三步讲清楚。

---

## 一、代码中 0 号点出现在哪里

关键代码：

```cpp
memset(depth, 0x3f, sizeof depth);
depth[0] = 0;
depth[root] = 1;
```

还有：

```cpp
fa[j][k] = fa[fa[j][k - 1]][k - 1];
```

以及：

```cpp
if (depth[fa[a][k]] >= depth[b])
```

这里 **都可能访问到 `fa[x][k] = 0`**。

所以需要：

```
0 号点存在
```

并且：

```
depth[0] = 0
```

---

## 二、为什么必须有 0 号点

看这一句：

```cpp
fa[j][k] = fa[fa[j][k - 1]][k - 1];
```

假设：

```
j = root
```

那么

```
fa[root][0] = 0
```

因为根节点没有父亲。

于是：

```
fa[root][1]
=
fa[fa[root][0]][0]
=
fa[0][0]
```

如果 **没有0号点**：

```
fa[0][0] 会越界
```

程序就会出错。

---

## 三、0号点相当于“空父节点”

逻辑结构其实是这样：

```
        0
        |
      root
     /   \
    2     3
```

但注意：

```
0 不是真实节点
```

只是表示：

```
没有父节点
```

---

## 四、举个完整例子

假设树：

```
    1
   / \
  2   3
```

root = 1

父节点表：

|节点|fa[x][0]|
|---|---|
|1|0|
|2|1|
|3|1|

---

继续计算倍增祖先：

```
fa[2][1] = fa[1][0] = 0
fa[2][2] = fa[0][1] = 0
```

```
fa[3][1] = fa[1][0] = 0
```

所以祖先表：

|x|2⁰|2¹|2²|
|---|---|---|---|
|1|0|0|0|
|2|1|0|0|
|3|1|0|0|

---

## 五、为什么 depth[0] = 0

看这段代码：

```cpp
if (depth[fa[a][k]] >= depth[b])
```

如果

```
fa[a][k] = 0
```

就会比较：

```
depth[0] >= depth[b]
```

而：

```
depth[0] = 0
```

通常

```
depth[b] ≥ 1
```

所以条件 **不会成立**，程序就不会继续跳。

这就避免了：

```
跳到不存在的祖先
```

---

## 六、总结一句话

LCA中的 **0号点作用**：

```
1 防止访问 fa[0][k] 越界
2 作为根节点的虚拟父节点
3 让倍增公式统一成立
4 避免写额外判断
```

本质是一个：

```
哨兵节点 (sentinel node)
```

---

## 七、记忆口诀

写 LCA 时可以记：

```
根的父亲是0
0的深度是0
0的祖先还是0
```

这样：

```
倍增永远不会越界
```

# 代码

```c++
#include<bits/stdc++.h>
using namespace std;

// N：最大节点数
// M：最大边数（无向树需要×2）
const int N = 4e4 + 10, M = N * 2;

// n：节点数量
// m：查询次数
// root：树的根节点
int n, m, root;

// fa[x][k]：节点x的第2^k级祖先
// depth[x]：节点x的深度
// st[x]：BFS访问标记
int fa[N][16], depth[N], st[N];

// 链式前向星存图
// h[x]：x的第一条边
// e[i]：边i指向的节点
// ne[i]：下一条边
// idx：边编号
int h[N], e[M], ne[M], idx;


// 加边函数：a -> b
void add(int a,int b){
    e[idx] = b;
    ne[idx] = h[a];
    h[a] = idx++;
}


// BFS遍历整棵树
// 作用：
// 1 计算每个节点的深度
// 2 预处理每个节点的倍增祖先
void bfs(){

    queue<int> q;

    // 从根节点开始
    q.push(root);

    depth[root] = 1;  // 根深度为1
    st[root] = 1;     // 标记访问

    while(!q.empty()){

        int u = q.front();
        q.pop();

        // 遍历u的所有邻居
        for(int i = h[u]; i != -1; i = ne[i]){

            int j = e[i];

            // 如果没有访问过
            if(!st[j]){

                depth[j] = depth[u] + 1;  // 更新深度
                st[j] = 1;
                q.push(j);

                // 记录父节点
                fa[j][0] = u;

                // 计算倍增祖先
                // fa[j][k] = j的2^k祖先
                for(int k = 1; k <= 15; k++)
                    fa[j][k] = fa[fa[j][k-1]][k-1];
            }
        }
    }
}


// 求两个节点的最近公共祖先
int lca(int a,int b){

    // 保证a更深
    if(depth[a] < depth[b])
        swap(a,b);

    // 第一步：让a跳到和b同一深度
    for(int k = 15; k >= 0; k--){
        if(depth[fa[a][k]] >= depth[b])
            a = fa[a][k];
    }

    // 如果此时相同
    // 说明其中一个是祖先
    if(a == b) return a;

    // 第二步：同时向上跳
    // 找到刚好不同的位置
    for(int k = 15; k >= 0; k--){
        if(fa[a][k] != fa[b][k]){
            a = fa[a][k];
            b = fa[b][k];
        }
    }

    // 返回它们的父节点
    // 就是LCA
    return fa[a][0];
}


int main(){

    cin >> n;

    // 初始化邻接表
    memset(h,-1,sizeof h);

    // 输入树结构
    while(n--){

        int a,b;
        cin >> a >> b;

        // b=-1说明a是根
        if(b == -1){
            root = a;
            continue;
        }

        // 建立无向边
        add(a,b);
        add(b,a);
    }

    // BFS预处理
    bfs();

    // 查询次数
    cin >> m;

    while(m--){

        int a,b;
        cin >> a >> b;

        // 求LCA
        int p = lca(a,b);

        // 判断祖先关系
        if(p == a){
            cout << 1 << endl;   // a是b祖先
        }
        else if(p == b){
            cout << 2 << endl;   // b是a祖先
        }
        else
            cout << 0 << endl;   // 都不是
    }
}
```
