# 思路

https://www.acwing.com/solution/content/5623/

这段代码解决的是经典的**数位 DP（动态规划）**问题：统计区间 $[a, b]$ 中每个数字（0-9）出现的次数。

它的核心思想是：**与其直接统计区间，不如分别计算 $1 \sim n$ 中每个数字出现的次数，然后利用前缀和思想 $count(b) - count(a-1)$ 得到结果。**

---

### 核心逻辑：分段统计法

假设我们要统计数字 $x$ 在 $n$ 中出现的次数。我们将 $n$ 拆解，观察每一位（假设为第 $i$ 位，从右往左数）上 $x$ 出现的频率。

假设数字 $n$ 为 `abc d efg`（其中 $d$ 是当前正在统计的那一位）：

#### 情况 1：前缀部分（abc）小于当前上限

当左边的数字范围在 `000` 到 `abc - 1` 之间时：

- **d 这一位固定为 x**。
    
- **右边 efg 部分**：可以取 `000` 到 `999`，共 $10^i$ 种可能（$i$ 为 $d$ 后面的位数）。
    
- **左边 abc 部分**：可以取 `000` 到 `abc - 1`，共 $abc$ 种可能。
    
- **结果**：$abc \times 10^i$。
    

> **注意特例 x = 0**：如果 $x=0$，左边不能从 `000` 开始（因为不能有前导 0），所以要少一种情况，结果为 $(abc - 1) \times 10^i$。

---

#### 情况 2：前缀部分（abc）正好等于当前上限

这时取决于当前位 $d$ 与 $x$ 的关系：

1. **如果 $d < x$**：
    
    无论后面怎么变，这一位都不可能出现 $x$（因为已经超过了 $n$ 的范围）。
    
    - **结果**：$0$。
        
2. **如果 $d == x$**：
    
    这一位可以是 $x$，但右边受限制，只能取 `000` 到 `efg`。
    
    - **结果**：$efg + 1$。
        
3. **如果 $d > x$**：
    
    这一位取 $x$ 时，左边已经是 `abc`（上限），由于 $d > x$，后面 `efg` 可以随便取 `000` 到 `999`。
    
    - **结果**：$10^i$。
        

---

### 举个例子：计算 $1 \sim 1234$ 中 $1$ 出现的次数

我们来看看**百位**（即 $d=2$）上 $1$ 出现了多少次：

此时 $n = 1234$，拆分为：$abc=1, d=2, efg=34$。我们要统计 $x=1$。

1. **情况 1（左边小于上限）**：
    
    左边取 $0$（即 `001 - 099` 这种形态）。
    
    此时 $d$ 固定为 $1$，后面两位可以取 $00 \sim 99$。
    
    数量：$1 \times 10^2 = 100$ 次。
    
2. **情况 2（左边等于上限）**：
    
    左边取 $1$。
    
    观察当前位 $d=2$。因为 $d(2) > x(1)$，满足情况 2.3。
    
    说明百位取 $1$ 时，后面可以从 $00 \sim 99$ 随便取。
    
    数量：$10^2 = 100$ 次。
    

**总计**：百位上 $1$ 出现了 $200$ 次。

---

### 代码关键点解析

- **`get(num, l, r)`**：这是从 `num` 数组中把那一串数字“抠”出来。比如 `1234` 存成 `[4, 3, 2, 1]`，`get(num, 3, 1)` 得到的就是 `123`。
    
- **`for(int i = n - 1 - !x; i >= 0; i -- )`**：
    
    - 如果我们要统计的数字 $x \neq 0$，从最高位开始看。
        
    - 如果 $x = 0$，**必须从次高位开始看**（因为最高位不能是 0，题目要求不计前导 0）。
        
- **`res += get(num, n - 1, i + 1) * power10(i)`**：这就是情况 1 的计算逻辑。
    

---
我们拿一个具体的数字来**手动模拟**一遍。

假设我们要计算 **1 到 123** 中数字 **1** 出现了多少次。即调用 `count(123, 1)`。

### 1. 准备阶段

代码将 `123` 倒序存入 `vector<int> num`：

`num[0] = 3`, `num[1] = 2`, `num[2] = 1`。

此时 `n = 3` (三位数)。

---

### 2. 进入循环：逐位统计

我们要统计 $x=1$ 在每一位上出现的次数。

#### **第一轮：统计百位 (i = 2)**

此时 `num[2] = 1`。

- **情况 1 (i < n-1)**：不满足（百位就是最高位，前面没有数字）。
    
- **情况 2.2 (num[i] == x)**：满足！因为 `num[2] == 1`。
    
    - 结果加上 `get(num, 1, 0) + 1`。
        
    - `get(num, 1, 0)` 是取低位数字，即 `23`。
        
    - **res = 23 + 1 = 24**。
        
    
    > **直观理解**：在百位取 1 时，后面可以取 `00` 到 `23`，共 24 个数（100, 101, ..., 123）。
    

---

#### **第二轮：统计十位 (i = 1)**

此时 `num[1] = 2`。

- **情况 1 (前缀影响)**：满足 `i < n-1`。
    
    - `res += get(num, 2, 2) * power10(1)`。
        
    - `get(num, 2, 2)` 是百位数字 `1`。`power10(1)` 是 `10`。
        
    - **res = 24 + (1 * 10) = 34**。
        
    
    > **直观理解**：当前缀（百位）取 `0` 时（即数字 010-019），十位可以固定为 1，个位 0-9 随便排。
    
- **情况 2.3 (num[i] > x)**：满足！因为 `num[1](2) > 1`。
    
    - `res += power10(1)`，即加上 `10`。
        
    - **res = 34 + 10 = 44**。
        
    
    > **直观理解**：当前缀取上限 `1` 时（即数字 110-119），因为十位 `2` 大于我们要找的 `1`，所以十位取 1 的情况全部跑满了。
    

---

#### **第三轮：统计个位 (i = 0)**

此时 `num[0] = 3`。

- **情况 1 (前缀影响)**：满足 `i < n-1`。
    
    - `res += get(num, 2, 1) * power10(0)`。
        
    - `get(num, 2, 1)` 是前缀 `12`。`power10(0)` 是 `1`。
        
    - **res = 44 + (12 * 1) = 56**。
        
    
    > **直观理解**：前缀取 `00` 到 `11` 时，个位固定为 1。共 12 种情况。
    
- **情况 2.3 (num[i] > x)**：满足！因为 `num[0](3) > 1`。
    
    - `res += power10(0)`，即加上 `1`。
        
    - **res = 56 + 1 = 57**。
        
    
    > **直观理解**：前缀取上限 `12` 时，因为个位 `3` 大于 `1`，所以 `121` 这个数也符合要求。
    

---

### 3. 最终结果

`count(123, 1)` 返回 **57**。

### 总结表格

|**位数 (i)**|**当前位数字**|**统计数字 x**|**计算逻辑**|**本位贡献次数**|
|---|---|---|---|---|
|百位 (2)|1|1|`23 + 1`|24|
|十位 (1)|2|1|`(1 * 10) + 10`|20|
|个位 (0)|3|1|`(12 * 1) + 1`|13|
|**总计**||||**57**|

### 关于 $x=0$ 的特殊处理

代码中有一句 `if(!x) res -= power10(i);`。

这是因为如果我们要找的是 **0**，在情况 1 中，前缀不能全为 0（比如数字 `005`，我们不认为它百位和十位有 0，它只是 `5`）。所以要减去那一种前导零的情况。

统计 $x = 0$ 是这段代码中最难理解的部分，因为在数学和编程习惯中，我们通常不计**前导零**。比如数字 `5`，我们不会说它是 `005`，所以不会认为它百位和十位有 `0`。

代码中通过 `i = n - 1 - !x` 和 `if(!x) res -= power10(i)` 这两处逻辑巧妙地避开了前导零。

---

### 1. 举个例子：计算 $1 \sim 123$ 中 $0$ 出现的次数

调用 `count(123, 0)`：

#### **第一步：确定起始位**

由于 `!x` 为真（$x=0$），循环起始位 `i = 3 - 1 - 1 = 1`。

- **这意味着：** 统计 $0$ 时，直接**跳过最高位**。因为最高位如果是 $0$，这个数就不是三位数了。
    

#### **第二轮：统计十位 (i = 1)**

此时 `num[1] = 2`（数字是 `1 2 3`）。

- **情况 1（前缀影响）**：`i < n-1` 成立。
    
    - `res += get(num, 2, 2) * power10(1)` $\rightarrow$ `1 * 10 = 10`。
        
    - **特殊处理 `if(!x)`**：`res -= power10(1)` $\rightarrow$ `10 - 10 = 0`。
        
    - **为什么？** 因为当百位（前缀）取 `0` 时，形成的数字是 `00x` 到 `009`，这些是前导零，不能计入。
        
- **情况 2.3 (num[1] > 0)**：满足 `2 > 0`。
    
    - `res += power10(1)` $\rightarrow$ `10`。
        
    - **当前 res = 10**。
        
    
    > **直观理解**：这 10 个数是 `100, 101, 102, ..., 109`。这些数在十位上有 `0`。
    

#### **第三轮：统计个位 (i = 0)**

此时 `num[0] = 3`。

- **情况 1（前缀影响）**：`i < n-1` 成立。
    
    - `res += get(num, 2, 1) * power10(0)` $\rightarrow$ `12 * 1 = 12`。
        
    - **特殊处理 `if(!x)`**：`res -= power10(0)` $\rightarrow$ `12 - 1 = 11`。
        
    - **为什么？** 前缀可以取 `01, 02, ..., 12`。减去的那个 `1` 是当前缀为 `00` 的情况（即数字 `000`）。
        
- **情况 2.3 (num[0] > 0)**：满足 `3 > 0`。
    
    - `res += power10(0)` $\rightarrow$ `1`。
        
    - **最终 res = 10 + 11 + 1 = 22**。
        

---

### 2. 核心逻辑总结

针对 $x=0$，代码做了两件事：

1. **避开最高位**：`i` 从 `n-2` 开始，保证了像 `012` 这种最高位为 `0` 的情况根本不进入计算。
    
2. **减去前导零**：在统计中间位时，虽然前缀（左边数字）可以从 `0` 开始取，但如果前缀全为 `0`，那么当前位的 `0` 依然是前导零。
    

|**场景**|**计算逻辑意义**|
|---|---|
|**跳过最高位**|排除 `0xx` 这种形式的数字。|
|**res -= power10(i)**|排除 `000...0`（前缀全零）导致当前位变成前导零的情况。|

---

### 3. 为什么 $x=0$ 的计算这么绕？

我们可以直观验证一下 $1 \sim 123$ 中有多少个 $0$：

- **10, 20, 30, ..., 90** (9个)
    
- **100, 101, ..., 109** (11个，注意100有两个0)
    
- **110, 120** (2个)
    
- **总计**：$9 + 11 + 2 = 22$ 个。

# 代码
```c++
#include <iostream>
#include <algorithm>
#include <vector>

using namespace std;
/*
分情况讨论：
abc d efg 中 x 出现的次数
1 最高位取 001 - abc - 1 时, d = x, 000 - 999
2.最高位取 abc
    2.1 d < x  无解：0
    2.2 d = x  000 - efg  总共 efg + 1种
    2.3 d > x  000 - 999  总共 1000   种
*/


int get(vector<int> num, int l, int r)
{
    int res = 0;
    // n = 1234  num 4 3 2 1
    // 注意因为 num 是倒序存储的各位数字，所以相当于是从后往前加
    for(int i = l; i >= r; i -- )
        res = res * 10 + num[i];
    return res;
}

// 计算出 10 ^ x 
int power10(int x)
{
    int res = 1;
    while( x -- ) res *= 10;
    return res;
}

int count(int n, int x)
{
    if(!n) return 0;
    // 用 vector 来倒序存储 n 例如 1423 存储的顺序是 3 2 4 1
    vector<int> num;
    while(n)
    {
        num.push_back(n % 10);
        n /= 10;
    }
    // n 用来表示位数
    n = num.size();

    int res = 0;
    // 从最高位 n - 1 开始判断，但是如果要判断的数是 0 的话，因为没有前导零，所以从第二位开始判断
    for(int i = n - 1 - !x; i >= 0; i -- )
    {
        //判断当前位之前的情况，所以前置条件是当前判断位不能是最高位
        if(i < n - 1)
        {   
            // 第 1 种情况
            res += get(num, n - 1, i + 1) * power10(i);
            // 特殊情况当 x = 0 时，不能有连续前导0，所以要去除 000 的情况
            if(!x) res -= power10(i);
        }
        // 2.1 种情况无解已经省略
        // 2.2 d == x
        if(num[i] == x) res += get(num, i - 1, 0) + 1;
        // 2.3 d > x
        else if(num[i] > x) res += power10(i);
    }

    return res;
}

int main()
{
    int a, b;
    while( cin >> a >> b, a || b )
    {
        if(a > b) swap(a, b);

        for(int i = 0; i < 10; i ++ )
            cout << count(b, i) - count(a - 1, i) << ' ';
        cout << endl;
    }

    return 0;
}
```
