# 题目

https://www.acwing.com/problem/content/1120/

给定 n 个正整数，将它们分组，使得每组中任意两个数互质。

至少要分成多少个组？

#### 输入格式

第一行是一个正整数 n。

第二行是 n 个不大于10000的正整数。

#### 输出格式

一个正整数，即最少需要的组数。

#### 数据范围

1≤n≤10

#### 输入样例：

```
6
14 20 33 117 143 175
```

#### 输出样例：

```
3
```

# 思路

这道题是经典的搜索题——**“分成互质组”**。

**核心目标**：给定 $n$ 个正整数，将它们分成若干个组，使得每个组内的数字**两两互质**（最大公约数为 1）。求最少需要分成的组数。

---

### 1. 基础函数与全局变量

C++

```
const int N = 10; // 数据范围较小 (n <= 10)，这暗示了我们可以使用较深层的 DFS
int n;
int p[N];           // 存储输入的 n 个数
int group[N][N];    // group[g][gc] 表示第 g 组中的第 gc 个元素的下标
bool st[N];         // 标记数组：记录某个数是否已经被分到某个组里
int ans = N;        // 存储最少组数，初始化为最大可能值（每个数一组）

// 欧几里得算法：求 a 和 b 的最大公约数
int gcd(int a, int b)
{
    return b ? gcd(b, a % b) : a;
}

// 检查函数：判断第 i 个数能否放入当前的第 g 组
bool check(int group[], int gc, int i)
{
    // 遍历当前组内的所有数，如果 p[i] 与其中任何一个数不互质（gcd > 1），则不能放入
    for (int j = 0; j < gc; j ++ )
        if (gcd(p[group[j]], p[i]) > 1)
            return false;
    return true;
}
```

---

### 2. 核心搜索函数 `dfs`

这是一个**双层搜索**逻辑：既要搜“当前组能放哪些数”，也要搜“是否需要开新组”。

C++

```
// g: 当前正在处理第几组
// gc: 当前组内已经有的元素数量 (group count)
// tc: 整个棋盘已经处理掉的元素总数 (total count)
// start: 当前组为了避免重复组合，从哪个下标开始尝试放入数字
void dfs(int g, int gc, int tc, int start)
{
    // 最优性剪枝：如果当前组数已经 >= 已找到的最佳答案，没必要搜下去了
    if (g >= ans) return;
    
    // 递归出口：所有数字都分好组了，更新答案
    if (tc == n) {
        ans = g;
        return;
    }

    bool flag = true; // 标记：当前组是否还能再塞入新的数字
    
    // 尝试从 start 开始寻找可以放入当前组 g 的数字
    for (int i = start; i < n; i ++ )
        if (!st[i] && check(group[g], gc, i))
        {
            st[i] = true;
            group[g][gc] = i; // 将数字下标存入当前组
            
            // 继续在当前组 g 尝试放入下一个数字，更新 tc+1 和 start 为 i+1
            dfs(g, gc + 1, tc + 1, i + 1);
            
            st[i] = false; // 回溯
            flag = false;  // 只要放入了一个数，就说明当前组还没“封死”
        }

    // 如果遍历完后发现 flag 仍为 true，说明当前组 g 无法再放入任何剩余的数字了
    // 此时新开一组 g + 1，从 0 号元素开始重新筛选
    if (flag) dfs(g + 1, 0, tc, 0);
}
```

### 3. 主函数逻辑

C++

```
int main()
{
    cin >> n;
    for (int i = 0; i < n; i ++ ) cin >> p[i];

    // 从第 1 组开始搜，当前组内有 0 个元素，总共分好了 0 个元素，从下标 0 开始找
    dfs(1, 0, 0, 0);

    cout << ans << endl;
    return 0;
}
```

### 💡 核心考点总结

1. **互质判断**：
    
    利用 `gcd(a, b) == 1` 判断互质。注意，互质组的要求是**两两互质**，所以 `check` 函数需要遍历当前组内的**每一个**成员。
    
2. **搜索顺序与优化**：
    
    - **组合优化**：通过 `start` 变量，确保每一组内的数字是按原始顺序递增选取的，避免了 `(1, 2)` 和 `(2, 1)` 这种重复组合的搜索。
        
    - **贪心思想**：逻辑上优先尝试在“当前组”塞入尽可能多的数。如果塞不下了（`flag == true`），才被迫开新组。
        
3. **剪枝策略**：
    
    `if (g >= ans) return;` 这是一个非常高效的剪枝。一旦当前的分组方案已经不如之前的方案优，立刻停止这条路径。
    
这是一个非常敏锐的观察！你发现了搜索顺序中的一个核心逻辑：**“按组填充”** vs **“按数字选择”**。

在 yxc 的这段代码中，使用的是**按组填充**的逻辑。为了让你彻底明白，我们对比一下两种不同的 DFS 思路：

### 1. 为什么代码看起来只放“最后一组”？

在这段代码的逻辑里，我们并不是“不能”放前面的组，而是**当前的 DFS 状态本身就是“正在填充第 $g$ 组”**。

- **搜索的顺序是：** 选定第 1 组 $\rightarrow$ 把所有能塞进第 1 组的互质组合都试一遍 $\rightarrow$ 塞不下了，再开第 2 组 $\rightarrow$ 把剩下的数能塞进第 2 组的组合试一遍...
    
- **为什么不回头看前面的组？** 因为在进入第 $g$ 组的搜索之前，**前面的第 $1$ 到 $g-1$ 组已经“封箱”了**。搜索树的逻辑保证了：如果一个数字能放进前面的组，在之前的递归层级里已经尝试过了。
    

---

### 2. 两种 DFS 思路对比

#### 思路 A：按组填充（本题采用）

- **逻辑**：我手里有一个箱子，我扫一遍所有还没放进去的数，看看谁能放进来。
    
- **优点**：配合 `start` 变量，可以极大减少**组合重复**（即 $\{1, 2\}$ 和 $\{2, 1\}$ 被视为同一种情况）。
    
- **为什么快**：它利用了**组合数**的性质。它不关心某个数字具体在哪，只关心当前这个“坑”能不能填满。
    

#### 思路 B：按数字选择（你直觉中的逻辑）

- **逻辑**：我拿起数字 $i$，我遍历**所有已经开好的组**（从 1 到 $g$），看看能放进哪个；如果哪都放不下，我再开个新组。
    
- **伪代码逻辑：**
    
    C++
    
    ```
    void dfs(int num_idx, int group_count) {
        if (group_count >= ans) return; // 剪枝
        if (num_idx == n) { ans = group_count; return; }
    
        for (int i = 1; i <= group_count; i++) { // 遍历前面的所有组
            if (check(i, num_idx)) {
                put_into(i, num_idx);
                dfs(num_idx + 1, group_count);
                take_out(i, num_idx);
            }
        }
        // 或者新开一组
        dfs(num_idx + 1, group_count + 1);
    }
    ```
    
- **缺点**：这种方法会产生大量的**等效搜索**。比如数字 1 放在第 A 组、数字 2 放在第 B 组，和数字 1 放在第 B 组、数字 2 放在第 A 组，本质是一样的，但这种搜法会跑两次。
    

---

### 3. 代码中 `flag` 和 `dfs(g + 1, ...)` 的真正含义

看这段代码：

C++

```
if (flag) dfs(g + 1, 0, tc, 0);
```

这里的 `flag` 只有在“**剩余的所有数字都无法放入当前第 $g$ 组**”时才会是 `true`。

这并不意味着我们放弃了前面的数字，而是说：**对于当前这一种分组组合，第 $g$ 组已经尽力了，我们必须开启第 $g+1$ 组来消化剩下的数字。**

### 总结

你之所以觉得“不能放前面的组”，是因为代码的**视角**站在“组”这一层：

1. **第 1 层递归**：我们在为第一组挑人。
    
2. **第 2 层递归**：我们在第一组挑完人的基础上，为第二组挑人。
    
3. **第 $n$ 层递归**：所有人都有了归宿。
    

如果在第 2 层递归时，你试图把数字放回第一组，那其实是在**修改第 1 层递归的决策**。在 DFS 中，这种“修改”是通过**回溯（st[i]=false）**回到上一层来实现的，而不是在当前层往回插。

## gcd

**GCD** 是 **Greatest Common Divisor** 的缩写，中文叫做 **最大公约数**。

简单来说，如果有两个整数 $a$ 和 $b$，能够同时整除它们的最大正整数，就是它们的最大公约数。

---

### 1. 直观理解

假设有两个数 **12** 和 **18**：

- **12 的约数**有：1, 2, 3, 4, **6**, 12
    
- **18 的约数**有：1, 2, 3, **6**, 9, 18
    
- 它们共同的约数（公约数）有：1, 2, 3, 6
    
- 其中最大的那个是 **6**，所以 $gcd(12, 18) = 6$。
    

---

### 2. 核心算法：辗转相除法 (Euclidean Algorithm)

在程序设计中，我们不可能靠列举约数来找 GCD，最常用的是**欧几里得算法**。它的核心原理是：

> $a$ 和 $b$ 的最大公约数，等于 $b$ 和 $a \pmod b$ 的最大公约数。

用公式表示：

$$gcd(a, b) = gcd(b, a \pmod b)$$

**举个例子：求 $gcd(54, 21)$**

1. $54 \div 21 = 2 \dots 12$ （余数 12）
    
2. $21 \div 12 = 1 \dots 9$ （余数 9）
    
3. $12 \div 9 = 1 \dots 3$ （余数 3）
    
4. $9 \div 3 = 3 \dots 0$ （余数 0）
    
5. 当余数为 0 时，当前的除数 **3** 就是最大公约数。
    

---

### 3. 代码实现 (C++)

这是你之前代码中看到的那个递归函数，非常简洁：

C++

```
int gcd(int a, int b) {
    // 如果 b 为 0，说明上一步的 a % b 刚好整除，a 就是最大公约数
    // 否则继续递归计算 gcd(b, a % b)
    return b ? gcd(b, a % b) : a;
}
```

---

### 4. 几何意义

你可以把 GCD 想象成**地砖问题**：

如果你有一间长为 $a$、宽为 $b$ 的房间，你想用同一种**正方形地砖**铺满全屋且不切割地砖，那么这种地砖的**最大边长**就是 $gcd(a, b)$。

---

### 5. 为什么要学它？

在算法竞赛中，GCD 经常出现在以下场景：

1. **分数组：** 就像你刚才看的“分成互质组”，互质的定义就是 $gcd(a, b) = 1$。
    
2. **分数化简：** 分子分母同时除以 GCD 得到最简分数。
    
3. **最小公倍数 (LCM)：** 有一个重要公式：$lcm(a, b) = \frac{a \times b}{gcd(a, b)}$。
    
# 代码

```c++
#include<bits/stdc++.h>
using namespace std;

const int N = 11; // 因为 n 较小，可以使用较深层的搜索
int n, num[N], g[N][N], st[N], ans;
// num[N]: 存储输入的 n 个整数
// g[u][gc]: 第 u 组里的第 gc 个元素
// st[N]: 标记数组，st[i]=1 表示第 i 个数字已被分配到某个组
// ans: 记录当前找到的最小组数

// 欧几里得算法：求 a 和 b 的最大公约数
int gcd(int a, int b){
    return b ? gcd(b, a % b) : a;
}

/**
 * 检查数值 i 是否能放入当前组 g
 * @param g[]: 当前正在填充的组（一维数组）
 * @param gc: 当前组内已有元素的个数
 * @param i: 准备放入的数值
 * @return: 若与组内所有数互质则返回 true
 */
int check(int g[], int gc, int i){
    for(int j = 0; j < gc; j++){
        // 如果 gcd > 1，说明不互质，不能放入同一组
        if(gcd(g[j], i) > 1)
            return false;
    }
    return true;
}

/**
 * DFS 搜索函数
 * @param u: 当前正在处理第几组 (从 1 开始)
 * @param gc: 当前组 u 已经有多少个元素
 * @param tc: 整个棋盘总共已经处理了多少个元素
 * @param start: 当前组为了避免重复，从哪个下标开始挑选数字
 */
void dfs(int u, int gc, int tc, int start){
    // 【最优性剪枝】如果当前组数 u 已经达到或超过了已知的最小答案 ans，直接回溯
    if(ans <= u) return;

    // 【递归出口】如果所有数字都处理完了 (tc == n)，更新答案
    if(tc == n){
        ans = u; 
        return;
    }
    
    bool flag = true; // 标记：当前组 u 是否还能塞入更多数字
    
    // 尝试在当前组 u 中放入剩余的数字
    for(int i = start; i < n; i++){
        // 如果数字 i 未被使用，且能放入当前组 u
        if(!st[i] && check(g[u], gc, num[i])){
            st[i] = 1;              // 标记使用
            g[u][gc] = num[i];      // 放入当前组
            
            // 继续在当前组 u 寻找下一个能放进去的数
            dfs(u, gc + 1, tc + 1, i + 1); 
            
            st[i] = 0;              // 回溯：撤销标记
            flag = false;           // 只要放进去过一个数，说明当前组还没“封死”
        }
    }
    
    // 【新开一组】如果 flag 为 true，说明剩余所有数都不能放入组 u 了
    // 此时开启第 u + 1 组，重新从下标 0 开始找数字
    if(flag) dfs(u + 1, 0, tc, 0);
}

int main(){
    cin >> n;
    ans = n; // 初始化答案为 n (即每个数各占一组的情况)
    for(int i = 0; i < n; i++) cin >> num[i];
    
    // 从第 1 组开始，当前组 0 个元素，总共 0 个元素，从下标 0 开始搜
    dfs(1, 0, 0, 0);
    
    cout << ans << endl;
    return 0;
}
```
